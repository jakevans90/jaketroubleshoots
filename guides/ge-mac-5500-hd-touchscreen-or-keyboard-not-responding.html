GE Healthcare MAC 5500 HD

Touchscreen or Keyboard Not Responding

Asset Type

Electrocardiograph (EKG) Machine

Manufacturer

GE Healthcare

Model

MAC 5500 HD

What This Guide Helps With

Identifies common external causes of touchscreen or keyboard unresponsiveness, including power, cable connections, system freeze, calibration issues, or temporary software lockups.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

Confirm no patient is actively undergoing an EKG.

Expected outcome: Safe environment for troubleshooting.

2. Verify Device Power

Check that the machine is fully powered on and battery (if applicable) is charged.

Expected outcome: Power indicators active; device ready for input.

3. Inspect External Connections

Ensure keyboard, touchscreen cable (if detachable), and USB connections are secure.

Expected outcome: No loose cables causing input failure.

4. Perform a Soft Reset

Power cycle the unit by turning it off, waiting 30 seconds, and turning it back on.

Expected outcome: Temporary software glitches cleared; touchscreen/keyboard responsive.

5. Check for System Freeze or Error Messages

Observe the display for frozen screens or error alerts.

Expected outcome: Device returns to normal operation or displays actionable error code.

6. Confirm Calibration (Touchscreen Only)

Access calibration menu if touchscreen partially responds; recalibrate per on-screen instructions.

Expected outcome: Touch input correctly recognized across screen.

7. Swap Input Device (If External Keyboard Present)

If using an external keyboard, test with a known good keyboard.

Expected outcome: Confirms whether failure is keyboard-specific.

If the Problem Persists

All external causes have been ruled out.
Device should be:
Removed from service
Labeled Out of Service
Sent for repair or bench evaluation

Knowing when to stop prevents patient risk and further device damage.

Clinical Use Tip

Never troubleshoot while a patient is on the table.
Move patient to another device before addressing input issues.
Ensure all data is saved before power cycling.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Touchscreen and keyboard not responding during routine EKG setup."

Cause

What was observed during troubleshooting.

Example:
"Power confirmed, cables secure, software may be frozen; input devices tested."

Resolution

What action was taken.

Example:
"Soft reset performed; device partially responsive; sent for service evaluation."

Helpful Details to Include (If Known)

  • Power indicators
  • Error messages observed
  • Input device tested
  • Calibration status
  • Final device status (e.g., OOS)

Final Thought

Prioritize patient safety by halting procedures before troubleshooting. Start with logical external checks and escalate appropriately. Document all findings to maintain operational integrity and support repair workflow.

That is successful troubleshooting.
