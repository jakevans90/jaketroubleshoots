Philips IntelliVue MP Series

Alarm: Bad Measurement Server

Asset Type

Patient Monitor

Manufacturer

Philips

Model

IntelliVue MP Series

What This Guide Helps With

This guide helps troubleshoot the Bad Measurement Server INOP on a Philips IntelliVue MP Series patient monitor when the monitor detects an unsupported or incompatible Measurement Server connection. This condition can prevent connected parameters from displaying and may interrupt normal patient monitoring through the Measurement Server.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Confirm the monitor is not the only active source of patient monitoring before disconnecting the Measurement Server or cycling power.
  • If the patient is being actively monitored, have clinical staff transfer monitoring to another compatible monitor first.
  • Expected: Patient monitoring remains continuous and safe.
  • Why it matters: This alarm affects measurement server communication and can interrupt parameter monitoring.

Verify the Reported Alarm and Behavior

  • Confirm the exact INOP message is present: Bad Measurement Server.
  • Also confirm whether the monitor displays the prompt: Measurement Server not supported, unplug device, switch monitor off/on.
  • Observe whether the alarm appears immediately after connecting the Measurement Server, during monitor startup, after moving a server from another monitor, or after reconnecting bedside monitoring equipment.
  • Expected: The alarm is reproducible when the suspect Measurement Server is attached.
  • Why it matters: This confirms a true compatibility-related INOP rather than a loose connection or temporary communication fault.

Identify the Connected Measurement Server

  • Inspect the connected Measurement Server and confirm the model and revision.
  • This INOP is associated with an incompatible M3000A Measurement Server Revision A being connected to the IntelliVue MP Series monitor.
  • Expected: The connected server is identified by model and revision.
  • Why it matters: The monitor is reporting that the attached Measurement Server is not supported.

Disconnect the Unsupported Measurement Server

  • Remove the connected Measurement Server from the monitor.
  • Inspect the connector area for obvious damage, bent pins, contamination, or signs of forced insertion.
  • Expected: The unsupported server is removed and no connector damage is found.
  • Why it matters: The alarm cannot be cleared reliably while an incompatible server remains connected.

Power Cycle the Monitor

  • With the incompatible Measurement Server disconnected, switch the monitor off, wait briefly, and power it back on.
  • Confirm whether the INOP clears during startup.
  • Expected: The monitor starts without the Bad Measurement Server INOP when the unsupported server is disconnected.
  • Why it matters: The displayed prompt specifically directs removal of the unsupported device followed by monitor restart.

Connect a Compatible Measurement Server

  • Attach a known compatible Measurement Server or Multi-Measurement Server approved for use with the IntelliVue MP Series monitor.
  • Confirm that the server is fully seated and latched, if applicable.
  • Expected: The monitor recognizes the compatible server and restores available parameters.
  • Why it matters: This isolates the issue to Measurement Server compatibility rather than the monitor itself.

Check Whether the Problem Follows the Server

  • If available, test the suspect Measurement Server on an appropriate compatible monitor or test setup.
  • Do not continue using it on the IntelliVue MP monitor if it is identified as unsupported.
  • Expected: The issue follows the incompatible Measurement Server, not the bedside monitor.
  • Why it matters: This prevents unnecessary monitor repair when the cause is an unsupported attached device.

Verify Software / Model Compatibility if Needed

  • If the Measurement Server is not clearly labeled or there is confusion about supported versions, verify compatibility using site equipment records, model labeling, configuration documentation, or service support.
  • Expected: The connected Measurement Server is confirmed compatible before returning the monitor to use.
  • Why it matters: Similar-looking measurement servers may not be supported across all monitor generations or revisions.

Stop When Resolved

  • If removing the incompatible Measurement Server, power cycling the monitor, and connecting a compatible server clears the INOP, return the monitor to service after functional verification.
  • Confirm basic parameter recognition and normal alarm behavior before closing the work order.

If the Problem Persists

If the Bad Measurement Server INOP remains after removing the unsupported server, power cycling the monitor, and connecting a known compatible Measurement Server, external and common causes have been ruled out.

At that point, the monitor should be removed from service, labeled Out of Service, and sent for bench evaluation or repair.

Persistent INOP behavior with compatible accessories may indicate a monitor-side communication, connector, software, or internal hardware issue.

Knowing when to stop is proper troubleshooting.

Clinical Use Tip

Never troubleshoot Measurement Server compatibility on an active patient. Move the patient to a backup device first and maintain therapy and monitoring continuity before disconnecting servers, swapping accessories, or power cycling the monitor. An unsupported Measurement Server can prevent required parameters from displaying, which may delay recognition of patient changes.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Philips IntelliVue MP Series monitor displayed Bad Measurement Server INOP with prompt stating Measurement Server not supported, unplug device, switch monitor off/on. Parameter monitoring was unavailable through the connected server."

Cause

What was observed during troubleshooting.

Example:
"Incompatible Measurement Server connected. Suspect device identified as unsupported M3000A Measurement Server Revision A for this monitor configuration."

Resolution

What action was taken.

Example:
"Removed unsupported Measurement Server, power cycled monitor, connected compatible Measurement Server, and verified INOP cleared. Monitor recognized compatible server and returned to service."

Helpful Details to Include (If Known)

  • Alarm behavior, including the exact Bad Measurement Server INOP and whether the support prompt appeared
  • Accessories swapped, including Measurement Server model, revision, and whether a compatible server was tested
  • Power behavior, including whether the INOP cleared after the unsupported server was removed and the monitor was restarted
  • Environmental factors, including whether the issue occurred after moving servers between rooms, monitors, or care areas
  • Indicator lights, connector condition, latching condition, bent pins, contamination, or visible damage
  • Final device status, including whether the monitor returned to service or was labeled Out of Service for bench evaluation

Final Thought

The Bad Measurement Server alarm is best handled as a compatibility and configuration issue before assuming monitor failure. Confirm the message, identify the connected server, remove unsupported hardware, and verify operation with a compatible Measurement Server. Clear documentation helps prevent the same incompatible server from being reused on another monitor.

That is successful troubleshooting.
