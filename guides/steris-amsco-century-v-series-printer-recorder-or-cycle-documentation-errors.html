STERIS AMSCO Century V Series

Printer, recorder, or cycle documentation errors

Asset Type

Sterilizer

Manufacturer

STERIS

Model

AMSCO Century V Series

What This Guide Helps With

Identifies causes of failed printouts, missing cycle records, or documentation errors that prevent proper cycle traceability and regulatory compliance.

Step-by-Step Troubleshooting

Ensure Patient and Staff Safety

  • Confirm the sterilizer is not running a cycle and the chamber is depressurized. Documentation checks do not require chamber access, but safety must always come first.

Check Printer Power and Connections

  • Verify the printer or recorder is plugged in and powered on.
  • Check for loose cables or disconnected USB/network connections.
  • Expected outcome: Printer shows ready or idle status.

Verify Paper or Recording Media

  • Ensure the printer has paper, labels, or chart paper loaded correctly.
  • Look for jams or misfeeds.
  • Expected outcome: Paper feeds correctly without error.

Check Printer or Recorder Settings

  • Confirm the sterilizer’s recorder settings match the installed printer type.
  • Review date/time settings; incorrect timestamps can cause errors.
  • Expected outcome: Printer settings match device configuration.

Test a Manual Print

  • From the sterilizer interface, attempt a test print or cycle report print.
  • Observe if the print completes successfully.
  • Expected outcome: A test print generates without errors.

Check for Error Codes or Messages

  • Note any specific error codes on the sterilizer display.
  • Reference the user interface or service manual for code meaning.
  • Expected outcome: Identify if the error is external (paper, connection) or internal (recorder failure).

Swap or Reboot

  • If possible, replace cables or try a different printer if networked.
  • Power cycle the sterilizer and printer to clear temporary faults.
  • Expected outcome: Errors clear and printer functions normally.

If the Problem Persists

External issues have been ruled out. The recorder or printer may be internally failed.

  • Remove the sterilizer from service if required by policy.
  • Label as Out of Service for documentation purposes.
  • Escalate to repair or bench evaluation.

Knowing when to stop prevents further risk and wasted effort.

Clinical Use Tip

Never troubleshoot the printer or recorder while a cycle is active with instruments inside.

Ensure patient-critical cycles are completed and traceable using an alternate method if needed.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Sterilizer did not print cycle reports for multiple completed cycles."

Cause

What was observed during troubleshooting.

Example:
"Printer cable was loose and paper tray was empty."

Resolution

What action was taken.

Example:
"Secured cable, loaded chart paper, verified test print completed successfully."

Helpful Details to Include

  • Printer power indicator status
  • Cable connections checked
  • Paper or recording media presence
  • Any error codes displayed
  • Successful test print results
  • Device final status

Final Thought

Documentation is critical for patient safety and regulatory compliance. Always start with external, simple causes before escalating. Clear CCR entries support audit readiness and track resolution.

That is successful troubleshooting.
