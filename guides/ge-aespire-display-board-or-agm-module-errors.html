GE Healthcare Aespire

Display Board or AGM Module Errors

Asset Type

Anesthesia Machine

Manufacturer

GE Healthcare

Model

Aespire

What This Guide Helps With

Identifies causes for display malfunctions, touchscreen errors, or AGM module communication issues before patient exposure.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Confirm no patient is connected to the machine.
  • Move the patient to an alternate anesthesia workstation if ventilation is required.

Check Power Source

  • Verify the machine is plugged into a working, grounded outlet.
  • Confirm battery (if present) is charged.
  • Expected Outcome: Display powers on and remains stable.

Inspect External Connections

  • Ensure all cables to the display board and AGM module are secure.
  • Check for visible damage or loose connectors.
  • Expected Outcome: Proper communication between modules; error may clear.

Restart the Machine

  • Power cycle the anesthesia machine after confirming no patient is attached.
  • Expected Outcome: Temporary software or module glitches may resolve.

Check for Software Alerts or Messages

  • Review on-screen diagnostics or error logs for AGM module faults.
  • Expected Outcome: Identify whether issue is software or hardware related.

Swap AGM Module or Display (If Available)

  • If the machine has a spare AGM module or display, swap to test.
  • Expected Outcome: Confirms whether the fault is in the module/display.

If the Problem Persists

External causes have been ruled out. The display board or AGM module likely has an internal fault.

Remove the machine from service.

Label it Out of Service and send it for bench evaluation or manufacturer repair.

Knowing when to stop avoids patient risk and unnecessary troubleshooting.

Clinical Use Tip

Never attempt internal board repair during patient care.

Always move patients to a functioning machine first.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Display shows AGM module error during routine pre-use check."

Cause

What was observed during troubleshooting.

Example:
"Module communication failure confirmed; cables and power verified."

Resolution

What action was taken.

Example:
"Machine removed from service; sent for repair; alternate workstation used for patient."

Helpful Details to Include (If Known)

  • Power outlet tested
  • Cables inspected
  • Error codes or messages recorded
  • Module/board swap tested
  • Final device status: Out of Service

Final Thought

Patient safety is paramount. Follow logical troubleshooting, rule out external factors first, and escalate appropriately. Proper documentation ensures accountability and future reference.

That is successful troubleshooting.
