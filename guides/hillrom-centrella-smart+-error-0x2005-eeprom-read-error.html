Hillrom Centrella Smart+

Error: 0x2005 EEPROM Read Error

Asset Type

Hospital Bed

Manufacturer

Hillrom

Model

Centrella Smart+

What This Guide Helps With

Error 0x2005 EEPROM Read Error causing loss of stored settings, boot issues, or inconsistent system behavior due to memory read failure.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Remove the bed from patient use if functionality is affected (movement, controls, alarms)
  • Transfer the patient to a safe, working bed if needed

2. Verify the Reported Error and Behavior

  • Power on the bed and confirm Error 0x2005 is displayed
  • Observe when the error occurs (startup, during operation, intermittent)
  • Note symptoms such as loss of settings, abnormal behavior, or failed initialization

3. Perform a Full Power Cycle

  • Power off the bed
  • Disconnect AC power
  • Disconnect battery (if accessible externally)
  • Wait 60–90 seconds
  • Reconnect power and restart

4. Check Power Stability and Source

  • Verify the bed is plugged into a known-good outlet
  • Inspect the power cord for damage
  • Confirm there is no intermittent power loss

5. Inspect External Cable Connections

  • Check visible cable connections between control components (no disassembly)
  • Ensure connectors are fully seated and secure

6. Observe Error Behavior Pattern

  • Restart the bed multiple times
  • Determine if the error is intermittent or persistent

7. Determine Resolution

  • If error clears and does not return, return bed to service
  • Monitor for recurrence

If the Problem Persists

External causes have been ruled out. EEPROM read errors indicate internal memory or control board failure.

  • Remove device from service
  • Label Out of Service
  • Replace the System Control Board (SCB) or send for repair

Clinical Use Tip

Never troubleshoot on an active patient. Move the patient to a backup device first before addressing memory-related errors.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Bed displaying Error 0x2005 EEPROM Read Error during startup; unit intermittently losing settings and failing to initialize properly."

Cause

What was observed during troubleshooting.

Example:
"Internal EEPROM read failure likely due to defective System Control Board after power cycle and external checks did not resolve issue."

Resolution

What action was taken.

Example:
"Removed bed from service, labeled out of service, and scheduled SCB replacement. Verified stable power and performed reset prior to escalation."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

EEPROM errors are critical indicators of memory failure. Always rule out power and connection issues first, but escalate appropriately when hardware failure is confirmed.

That is successful troubleshooting.
