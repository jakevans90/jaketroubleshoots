Natus Olympic Brainz Monitor

Printer / Data Export Failure

Asset Type

EEG System

Manufacturer

Natus

Model

Olympic Brainz Monitor

What This Guide Helps With

Troubleshooting printer, report printing, USB, network, or data export failures on the Olympic Brainz Monitor.

Step-by-Step Troubleshooting

Ensure Patient Safety First

Confirm the EEG monitoring session is not being interrupted for an active patient before troubleshooting printing or export functions.

If the monitor is actively being used for patient monitoring, do not power cycle, disconnect accessories, or remove storage media unless clinical staff confirms it is safe.

Expected outcome: Patient monitoring remains uninterrupted while non-urgent printer or export issues are assessed.

Verify the Reported Failure

Ask the user what specifically failed:

  • Report would not print
  • Printer not detected
  • Export option unavailable
  • USB device not recognized
  • Export file missing or incomplete
  • Data transfer failed
  • Error message during print or export

Expected outcome: The complaint is narrowed to printer output, removable media, network transfer, or stored data access.

Confirm the Patient Data Is Still Viewable

Before attempting corrective actions, verify the required EEG/aEEG recording or report is still accessible on the monitor.

Do not delete studies, clear storage, or reformat media during basic troubleshooting.

Expected outcome: The needed patient recording is preserved before troubleshooting continues.

Check Printer Power and Basic Status

If an external printer is used, confirm it is powered on, has paper installed, has no paper jam, and shows no error condition.

Check for printer indicators such as offline status, low paper, low toner/ink, or fault lights.

Expected outcome: Printer is ready and not failing due to a basic supply or status issue.

If this resolves the issue, document the printer condition and stop.

Check Printer Cable or Connection

Inspect the printer connection between the Olympic Brainz Monitor and printer.

Confirm USB, serial, or network cables are fully seated and not damaged, stretched, or loose.

If available, reseat the cable and test with a known-good cable.

Expected outcome: The printer connection is secure and communication can be reattempted.

If printing resumes after reseating or replacing the cable, return the system to service if no other issues are present.

Verify the Correct Printer Is Selected

Check the monitor’s print or report output settings, if accessible.

Confirm the selected printer or output destination matches the connected printer.

Expected outcome: The monitor is not attempting to print to an unavailable or incorrect destination.

Test Printer Function Outside the Monitor if Possible

If the printer can be tested independently, print a self-test page or connect it to another approved workstation.

Expected outcome: The printer itself is confirmed functional.

If the printer fails independent testing, remove or replace the printer rather than assuming the EEG monitor has failed.

Check USB Storage Media for Data Export

If the issue involves USB export, confirm the USB device is properly inserted and recognized.

Use facility-approved removable media only.

Inspect the USB drive for physical damage, bent connector contacts, or signs of wear.

Expected outcome: The USB device is physically intact and properly connected.

Try a Known-Good USB Device

Attempt export using a known-good, approved USB drive.

Avoid using encrypted, damaged, unusually large, or unknown-format USB drives unless supported by facility policy and the device workflow.

Expected outcome: Export succeeds with known-good media.

If export works with another USB device, the original USB drive should be removed from use or reformatted/replaced according to IT and facility policy.

Check Available Storage and File Selection

Confirm the correct patient study, report, time range, or export option was selected.

Verify the export destination has enough available space.

Expected outcome: The export is not failing because of an incorrect selection or lack of storage space.

Look for Error Messages or Export Codes

Record any displayed message exactly as shown.

Note whether the failure occurs immediately, during file preparation, during transfer, or after the export appears complete.

Expected outcome: The failure pattern is documented for repair, IT, or vendor support if escalation is needed.

Check Network Connection if Export Uses Network Transfer

If the Olympic Brainz Monitor exports through a network location or hospital system, verify the network cable is connected or wireless/network status is normal, depending on the installed configuration.

Check whether other devices in the same area are having network issues.

Expected outcome: A local network or hospital IT issue is ruled in or ruled out.

Confirm Date, Time, and Patient Identification

Check that the device date/time and patient/study information appear correct.

Incorrect study selection or mismatched patient information can lead staff to believe an export failed when the file was saved under a different identifier.

Expected outcome: The exported file can be matched to the correct patient record or study.

Restart Only When Clinically Safe

If the monitor is not actively being used for patient monitoring, perform a normal shutdown and restart.

Do not force power removal unless the unit is locked up and facility procedure allows it.

Expected outcome: Temporary software or communication issues clear after a controlled restart.

If printing or export works after restart, document the action and verify normal operation before returning to use.

Perform a Functional Check

After corrective action, perform a test print or test export using non-patient or approved test data when possible.

Confirm the file opens or the report prints correctly.

Expected outcome: Printer or export function is verified before the system is returned for clinical use.

If the Problem Persists

If printer power, cables, printer status, USB media, storage capacity, export selection, network connection, and safe restart have all been checked, the issue may involve internal software, communication hardware, storage failure, configuration corruption, or printer interface failure.

Remove the Olympic Brainz Monitor from service if the failure affects required clinical documentation, patient record transfer, or reliable operation.

Label the device Out of Service and send it for bench evaluation, IT review, or vendor repair as appropriate.

Knowing when to stop is proper troubleshooting. Do not attempt deep internal disassembly, software reloads, or unsupported data recovery during floor-level troubleshooting.

Clinical Use Tip

Do not troubleshoot printing or data export by interrupting active EEG monitoring unless clinical staff confirms monitoring can safely stop. Preserve patient data first, then troubleshoot the output problem.

If the monitor is needed for active patient monitoring, move the patient to a backup device or preserve therapy continuity according to clinical procedure before troubleshooting continues.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Clinical staff reported the Natus Olympic Brainz Monitor would not export EEG recording data to USB after patient monitoring was completed."

Cause

What was observed during troubleshooting.

Example:
"Verified the original USB drive was not recognized by the monitor; export completed successfully using a known-good approved USB drive."

Resolution

What action was taken.

Example:
"Replaced the faulty USB media, confirmed successful test export, and returned the Olympic Brainz Monitor to service."

Helpful Details to Include (If Known)

  • Whether the issue was print, USB export, or network export
  • Exact error message displayed
  • Patient data still viewable on monitor
  • Printer power and paper status checked
  • Printer cable reseated or swapped
  • USB drive swapped with known-good media
  • Export file name or study identifier verified
  • Network connection checked
  • Date and time verified
  • Restart performed or not performed
  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Printer and data export problems should be approached carefully because the main risk is loss of clinical documentation, not just failed output. Start with the printer, cable, USB media, storage, and network path before suspecting monitor failure. Protect patient data, avoid unnecessary disruption to active monitoring, and document the complaint, cause, and resolution clearly.

That is successful troubleshooting.
