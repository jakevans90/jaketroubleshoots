STERIS STERIS AMSCO 4085

Error: KEY ONLY

Asset Type

Surgical Table

Manufacturer

STERIS

Model

STERIS AMSCO 4085

What This Guide Helps With

KEY ONLY error indicating multiple buttons pressed simultaneously, causing control lockout or unintended table movement restrictions.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • If the table is in use, stop movement immediately
  • Stabilize the patient before troubleshooting
  • Transfer patient if control is unreliable

Verify the Reported Error and Behavior

  • Observe the display for the “KEY ONLY” message
  • Attempt to press a single function button
  • Check if controls are unresponsive or intermittent

Check for Multiple Button Presses

  • Ensure only one button is pressed at a time
  • Ask staff if both hand and foot controls were used
  • Release all buttons and retry a single input

Inspect for Stuck or Jammed Buttons

  • Press each button individually
  • Look for stuck, damaged, or sunken buttons
  • Check for debris or fluid contamination

Isolate Control Source (Hand vs Foot Control)

  • Disconnect the foot control and test hand control
  • Reconnect and test foot control separately
  • Identify if one control triggers the error

Check Cable Connections

  • Ensure all cables are fully seated
  • Inspect for fraying or bent pins
  • Move cables gently to check for intermittent faults

Restart the Table System

  • Power the table off completely
  • Wait 10–15 seconds
  • Power back on and retest controls

If the Problem Persists

If the KEY ONLY error continues after external checks:

  • Internal control board or input circuitry may be faulty
  • Remove the table from service
  • Label as Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot table controls on an active patient. Move the patient to a backup device before testing inputs.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Table displaying 'KEY ONLY' error and not responding properly to control inputs."

Cause

What was observed during troubleshooting.

Example:
"Simultaneous input detected due to stuck foot control button causing continuous signal."

Resolution

What action was taken.

Example:
"Disconnected faulty foot control, verified normal operation with hand control, and removed foot control from service."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

The KEY ONLY error is typically a safety feature preventing conflicting commands. Focus on simple input issues first, as most are resolved without invasive repair. Proper isolation and documentation ensure safe and efficient troubleshooting.

That is successful troubleshooting.
