Stryker SDC Ultra

Touchscreen or Soft Key Failure

Asset Type

Endoscopic / OR Camera System

Manufacturer

Stryker

Model

SDC Ultra

What This Guide Helps With

Unresponsive touchscreen or soft keys, delayed input, or incorrect responses caused by external factors, connections, or interface issues.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • If the system is being used during a procedure, notify staff and transition to an alternate control method or backup system.
  • Expected: Procedure continues safely
  • Why it matters: Loss of control can delay documentation or interfere with workflow during procedures

Verify the Reported Issue

  • Attempt to use the touchscreen and soft keys.
  • Observe:
    • No response to touch
    • Intermittent or delayed response
    • Incorrect selections being triggered
  • Expected: Issue is reproducible
  • Why it matters: Confirms this is a true interface problem, not user error

Check for Screen Lock or Software State (Common Cause)

  • Verify the system is not locked or in a restricted mode.
  • Action:
    • Look for lock icons or prompts
    • Attempt to exit any menus or overlays
    • Restart the user interface if accessible
  • Expected: Normal control restored if caused by software lock
  • Why it matters: Locked or frozen UI is a frequent cause of perceived failure

Perform a System Restart (Very Common Fix)

  • Power cycle the SDC Ultra.
  • Action:
    • Shut down the system properly if possible
    • Disconnect power for 10–15 seconds
    • Power back on
  • Expected: Touchscreen and soft keys respond normally after reboot
  • Why it matters: Clears temporary software or interface glitches

Inspect Touchscreen Surface

  • Check for anything interfering with touch input.
  • Action:
    • Clean the screen using approved wipes
    • Remove moisture, debris, or residue
    • Ensure no protective film is damaged or peeling
  • Expected: Improved or restored touch response
  • Why it matters: Contamination can block or distort touch input

Check for External Input Devices or USB Interference

  • Disconnect any external peripherals.
  • Action:
    • Remove USB drives, keyboards, or accessories
    • Re-test touchscreen and soft keys
  • Expected: Input control returns to normal
  • Why it matters: External devices can interfere with input priority or system responsiveness

Observe for System Performance Issues

  • Check if the system appears slow or frozen.
  • Action:
    • Look for lag, delayed menus, or system freeze
    • Check if recording or storage is near capacity
  • Expected: Normal system responsiveness
  • Why it matters: System overload or storage issues can affect UI responsiveness

Test Soft Keys Independently

  • If applicable, test physical soft keys separate from touchscreen input.
  • Action:
    • Press each soft key individually
    • Check for tactile feedback and response
  • Expected: Keys respond correctly
  • Why it matters: Helps differentiate between touchscreen failure vs control board or key interface issue

If the Problem Persists

If the touchscreen and soft keys remain unresponsive after these checks, external causes have been ruled out and the issue is likely internal (touch controller, display assembly, or system board).

Remove the device from service, label it Out of Service, and send for repair or bench evaluation. Proper troubleshooting includes recognizing when escalation is required.

Clinical Use Tip

Do not attempt to troubleshoot touchscreen issues during an active procedure. Always transition to a backup system before performing any troubleshooting steps.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Touchscreen and soft keys on SDC Ultra not responding during case setup."

Cause

What was observed during troubleshooting.

Example:
"Touchscreen unresponsive due to system software freeze."

Resolution

What action was taken.

Example:
"Power cycled system and verified normal touchscreen and soft key operation."

Helpful Details to Include (If Known)

  • Touchscreen completely unresponsive vs intermittent
  • Soft key response status
  • System freeze or lag observed
  • External devices connected
  • Screen condition (clean, damaged, wet)
  • Storage capacity or recording status
  • Final device status after reboot

Final Thought

Touchscreen and soft key failures are often caused by simple external or software-related issues. Always start with basic checks and resets before assuming hardware failure. Patient safety and clear documentation remain the priority throughout troubleshooting.

That is successful troubleshooting.
