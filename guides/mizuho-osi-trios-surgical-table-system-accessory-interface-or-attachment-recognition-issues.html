Mizuho OSI Trios Surgical Table System

Accessory Interface or Attachment Recognition Issues

Asset Type

Surgical Table

Manufacturer

Mizuho OSI

Model

Trios Surgical Table System

What This Guide Helps With

Identifies causes of accessory or attachment recognition failure, including connection, interface, power, or software issues before assuming internal electronics failure.

Step-by-Step Troubleshooting

Ensure Patient Safety

  • Confirm the table is not supporting a patient. No troubleshooting should occur while a patient is on the table.

Inspect Physical Connections

  • Check that all accessories and attachments are properly seated and locked in their ports. Loose or misaligned connectors often prevent recognition.

Check Interface Cables

  • Inspect cables connecting the table to accessory interfaces for damage, wear, or disconnection. Replace or reseat as needed.

Power Cycle the Table

  • Turn the table off, wait 30 seconds, and power it back on. Some recognition issues clear after a reset.

Verify Accessory Compatibility

  • Confirm that the attached accessory is compatible with the Trios system model. Non-compatible or defective attachments may not be recognized.

Observe Indicator Lights or Error Messages

  • Look for blinking lights, LEDs, or display notifications on the table and accessory. They can indicate specific connection or interface faults.

Swap Accessories or Ports

  • If possible, try another known-working accessory in the same interface, or the same accessory in another compatible port, to isolate the issue.

If the Problem Persists

  • After all external checks, compatible accessories, proper connections, and power resets have been verified, the issue is likely internal.
  • Remove the table from service
  • Label Out of Service
  • Send for repair or bench evaluation
  • Knowing when to escalate is proper troubleshooting.

Clinical Use Tip

  • Do not troubleshoot accessories on a table supporting a patient.
  • Move the patient to another table if needed.
  • Only perform recognition tests when safe.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Table does not recognize attached accessory during procedure."

Cause

What was observed during troubleshooting.

Example:
"Accessory port or interface cable loose, or accessory incompatible."

Resolution

What action was taken.

Example:
"Reseated connections, verified compatible accessory; issue persisted, sent for service evaluation."

Helpful Details to Include (If Known)

  • Which accessory/attachment was connected
  • Port or interface used
  • Indicator lights or error codes observed
  • Cables tested or swapped
  • Table software version
  • Final table status

Final Thought

Accessory recognition issues can often be resolved by verifying connections, power cycling, and confirming compatibility. Escalation should be timely to maintain patient safety and prevent procedure delays. Proper documentation ensures traceability and supports future troubleshooting.

That is successful troubleshooting.
