STERIS AMSCO 400 Series

Alarm: Input/Output Board #1 Communication Failure – Control board cannot communicate with I/O boards in control box

Asset Type

Sterilizer

Manufacturer

STERIS

Model

AMSCO 400 Series

What This Guide Helps With

This guide addresses the Input/Output Board #1 Communication Failure alarm, which causes a cycle abort due to loss of communication between the main control board and I/O boards. This failure prevents proper monitoring and control of the sterilization cycle.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Remove the sterilizer from service if the alarm occurs during a cycle
  • Do not use any load from an aborted cycle
  • Follow facility policy for reprocessing

2. Verify the Reported Alarm and Behavior

  • Confirm display shows “Input/Output Board #1 Communication Failure”
  • Observe if alarm occurs at startup or during cycle
  • Confirm cycle aborts automatically
  • Check for lack of progression through cycle steps

3. Perform Basic Reset and Power Check

  • Turn sterilizer OFF
  • Wait 30–60 seconds
  • Power unit back ON
  • Observe if alarm clears or immediately returns

4. Check External Power Stability

  • Verify unit is connected to stable facility power
  • Check for recent outages or voltage fluctuations
  • Confirm proper supply voltage if measurement is available

5. Inspect Control Cabinet (Non-Invasive Visual Check Only)

  • Open cabinet if permitted and safe
  • Look for loose or disconnected cables between boards
  • Check for partially seated boards or connectors
  • Do not interact with energized components

6. Reseat Accessible Connectors (If Within Scope)

  • Power OFF the unit
  • Reseat wiring harnesses between control board and I/O boards
  • Ensure connectors are fully engaged
  • Restore power and recheck alarm

7. Consider Configuration Issues

  • Determine if boards were recently replaced or serviced
  • Check for incorrect DIP switch configuration
  • Reference service manual if applicable

8. Attempt Functional Check

  • Restart unit and observe idle operation
  • Attempt safe test cycle if appropriate
  • Confirm no recurrence of alarm

If the Problem Persists

If the alarm continues after external checks, internal communication failure is likely.

  • Remove device from service
  • Label Out of Service
  • Send for qualified repair or bench evaluation

Clinical Use Tip

Never troubleshoot a sterilizer on an active patient load. Always move workflow to a backup sterilizer before attempting troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Sterilizer displayed Input/Output Board #1 Communication Failure and aborted cycle."

Cause

What was observed during troubleshooting.

Example:
"Loss of communication between control board and I/O boards; loose connection found or internal board failure suspected."

Resolution

What action was taken.

Example:
"Reseated connectors and restored operation — or removed unit from service and sent for repair."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Communication failures between control and I/O boards prevent safe sterilizer operation. Always rule out simple connection and power issues first, then escalate appropriately to ensure patient safety and system integrity.

That is successful troubleshooting.
