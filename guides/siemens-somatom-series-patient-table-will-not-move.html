Siemens Healthineers SOMATOM Series

Patient Table Will Not Move / Position Error

Asset Type

CT Scanner

Manufacturer

Siemens Healthineers

Model

SOMATOM Series

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting situations where the patient table on a Siemens Healthineers SOMATOM Series CT scanner will not move or generates a position error. Symptoms may include the table not responding to controls, stopping mid-movement, or the system reporting a table position fault during scanning or positioning.

This guide focuses on verifying external causes first, such as obstructions, control inputs, power issues, and safety interlocks, before assuming an internal table drive or positioning failure.

Step-by-Step Troubleshooting

1. Verify the System Is Fully Initialized

  • Ensure the CT system has completed startup and gantry initialization.
  • If the system is still booting or initialization failed, table movement may be disabled until the system reaches ready status.

2. Check the Table Control Panel

  • Attempt to move the table using the table control panel or operator console.
  • Confirm:
    • Table controls respond
    • Buttons are not physically stuck
    • No error messages appear on the console
  • If the console allows movement but the table does not respond, continue troubleshooting.

3. Check for Physical Obstructions

  • Inspect the full travel path of the table.
  • Look for:
    • Equipment carts
    • IV poles
    • Linens or patient straps caught in the rails
    • Debris or dropped objects near the base
  • Even small obstructions can trigger a safety stop or position error.

4. Verify Emergency Stop Is Not Activated

  • Check the CT system emergency stop buttons.
  • Confirm:
    • No emergency stop is engaged
    • All E-stop buttons are fully released
  • An engaged emergency stop disables table motion for safety.

5. Confirm Table Weight and Position Limits

  • Verify that the patient weight and positioning are within system limits.
  • Check for:
    • Excessive patient weight
    • Patient positioned too far forward or backward
    • Accessories improperly installed
  • If the system detects unsafe conditions, it may prevent table movement.

6. Inspect Table Cable Connections

  • Visually inspect accessible cables between the patient table and the gantry or base.
  • Look for:
    • Loose connectors
    • Pinched cables
    • Visible damage
  • Table movement depends on communication and power connections between the table and system controller.

7. Attempt Table Re-Zero or Reposition

  • If available from the system interface, attempt to:
    • Reset the table position
    • Return the table to home position
    • Reinitialize table positioning
  • Minor encoder or position errors may clear with a reposition command.

8. Power Cycle the Table System

  • If safe to do so and no patient is present:
    • Shut down the CT system according to procedure.
    • Wait 30–60 seconds.
    • Restart the system and allow full initialization.
  • This can clear temporary communication or positioning errors.

9. Observe for Mechanical Symptoms

  • During attempted movement, observe for:
    • Grinding or clicking noises
    • Jerky or partial movement
    • Table attempting movement but stopping immediately
  • These symptoms may indicate a mechanical drive issue or motor problem.

If the Problem Persists

If the patient table still will not move or continues reporting a position error after these checks, common external causes have likely been ruled out.

The issue may involve:

  • Table drive motor failure
  • Position encoder faults
  • Internal control electronics
  • Mechanical drive or rail issues

The device should be:

  • Removed from service
  • Labeled Out of Service
  • Scheduled for vendor service or detailed bench evaluation

Recognizing when internal repair is required is an important part of proper troubleshooting.

Clinical Use Tip

Do not troubleshoot table movement while a patient is on the CT table.

If the issue occurs during patient setup, safely assist the patient off the table and move them to another CT system if available before continuing troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
Staff reported the CT scanner patient table would not move when attempting to position the patient. The system displayed a table position error.

Cause

What was observed during troubleshooting.

Example:
Inspection revealed the emergency stop button on the gantry had been activated, preventing table movement.

Resolution

What action was taken.

Example:
Released the emergency stop, verified normal table movement using console controls, and confirmed the CT system returned to normal operation.

Helpful Details to Include

  • Emergency stop status
  • Table control panel response
  • Obstruction check completed
  • Cables inspected
  • System reboot performed
  • Any noises or partial movement observed
  • Final system status

Final Thought

Patient table movement issues on CT scanners often appear serious but are frequently caused by simple external conditions such as obstructions, safety interlocks, or initialization problems. A logical troubleshooting approach protects patient safety, reduces unnecessary service calls, and ensures accurate documentation for future reference.

That is successful troubleshooting.
