Philips Allura Xper

Table Movement Failure

Asset Type

Fluoroscopy / Interventional System

Manufacturer

Philips

Model

Allura Xper

What This Guide Helps With

Identifies external causes of table movement failure, such as power issues, interlocks, or mechanical obstructions, ensuring safe and functional patient positioning.

Step-by-Step Troubleshooting

Ensure Patient and Staff Safety

  • Confirm no patient is on the table and staff are clear.
  • Why: Prevents injury during table movement tests.

Check Table Power and Circuit Connections

  • Verify table is connected to a live power outlet and circuit breakers are on.
  • Check all table power cables for secure connections.
  • Expected outcome: Table electronics are powered and ready.

Inspect Emergency Stop and Interlocks

  • Ensure emergency stop buttons are released and safety interlocks (side rails, foot switches) are disengaged.
  • Expected outcome: Table is free to respond to movement commands.

Test Table Control Panel and Foot Switches

  • Operate table using both the main control panel and foot switch if available.
  • Expected outcome: Table responds to movement commands. If not, note which controls fail.

Check for Mechanical Obstructions

  • Inspect rails, casters, and underside of the table for debris or foreign objects.
  • Ensure cables under the table are not pinched.
  • Expected outcome: No physical blockages preventing movement.

Verify System Software Status

  • Check for system warnings or software errors that may inhibit table movement.
  • Reboot the system if safe and permitted by protocol.
  • Expected outcome: Software allows normal table operation.

If the Problem Persists

External power, interlocks, controls, and obstructions have been ruled out.

Action: Remove table from service, label as Out of Service, and send for vendor repair or bench evaluation.

Knowing when to stop is proper troubleshooting.

Clinical Use Tip

Do not attempt to move a table with a patient on it if failure occurs.

Move patients to a backup table or imaging system before further troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Table fails to move during standard positioning commands."

Cause

What was observed during troubleshooting.

Example:
"Emergency stop engaged and minor debris obstructing underside of table."

Resolution

What action was taken.

Example:
"Cleared obstruction, released emergency stop, and verified table movement via control panel and foot switch."

Helpful Details to Include

  • Emergency stop status
  • Interlock lights
  • Foot switch response
  • Audible table motor sounds
  • Any unusual resistance or binding during manual movement

Final Thought

Patient safety is paramount. Troubleshooting should progress logically from power and interlocks to mechanical checks. Document each step and escalate when external causes are excluded. Proper documentation ensures clear communication and supports efficient repair.

That is successful troubleshooting.
