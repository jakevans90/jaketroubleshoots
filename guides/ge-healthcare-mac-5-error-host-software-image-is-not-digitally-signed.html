GE Healthcare MAC 5

Error: Host Software Image Is Not Digitally Signed

Asset Type

Electrocardiograph (EKG) Machine

Manufacturer

GE Healthcare

Model

MAC 5

What This Guide Helps With

This guide helps troubleshoot the Host Software Image Is Not Digitally Signed error on the GE Healthcare MAC 5. This error prevents the software upgrade or installation from continuing because the host software image is not recognized as a valid digitally signed GE HealthCare software package.

The issue may be caused by an incorrect software image, incomplete software package, unofficial or unsupported file, corrupted file, renamed or modified file, wrong software package for the device, or software obtained from an unapproved source.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Remove the MAC 5 from patient use before attempting any software upgrade troubleshooting.
  • Do not perform software upgrade work while the cardiograph is needed for active ECG acquisition, patient testing, or clinical workflow.
  • Confirm the device is not connected to a patient and is available for safe bench troubleshooting.
  • Software upgrade failures can interrupt device function and should not occur during patient care.

Verify the Reported Error and Behavior

  • Confirm the exact displayed error is Host Software Image Is Not Digitally Signed.
  • Determine whether the error appears during USB software upgrade, network/shared-folder software upgrade, after selecting the host software image, or during upgrade validation before installation begins.
  • Confirm the error appears when the MAC 5 checks the host software image before allowing the upgrade to continue.
  • This confirms the issue is related to software image validation, not power, network, USB detection, or general system failure.

Translate the Error Into the Real-World Problem

  • Recognize that this error means the MAC 5 does not recognize the host software image as a valid digitally signed GE HealthCare software package.
  • Consider likely external causes such as incorrect software image, incomplete software package, unofficial file, unsupported file, corrupted file, renamed file, modified file, wrong device package, or software obtained from an unapproved source.
  • Do not treat this as a normal hardware failure or internal repair issue.
  • The device is blocking the upgrade to prevent installation of an unverified software image.

Check the Upgrade Source

  • If using USB, remove the USB flash drive and inspect the files on a computer.
  • Confirm the host software image is present and has not been renamed, altered, zipped incorrectly, or placed inside an unexpected folder.
  • Confirm the software image is present exactly as provided by GE HealthCare.
  • Changing the file name, file structure, or package contents can cause the system to reject the image.

Confirm the Software Came From an Approved Source

  • Ask where the software image came from.
  • Do not continue if the file was copied from another device, downloaded from an unofficial source, sent informally, or modified locally.
  • The software image should come from GE HealthCare Service Support or another approved GE HealthCare software distribution source.
  • The fix for this error is not to bypass the signature check. The correct fix is to obtain the properly signed software image.

Check for File Corruption or Incomplete Copy

  • If the file was copied to USB or a shared folder, delete the existing file from the media and copy it again from the original approved source.
  • Use a known-good USB flash drive or confirmed network shared folder if available.
  • Confirm the copied file matches the original approved software image and is fully transferred.
  • A partial or corrupted file may fail digital signature validation.

Verify the File Location

  • For USB upgrade, confirm the software image is in the required location on the USB flash drive, typically the root directory unless the approved upgrade instructions specify otherwise.
  • For shared-folder upgrade, confirm the valid signed image is placed in the configured shared network path.
  • Confirm the MAC 5 can locate and validate the correct software image.
  • A valid file in the wrong location may cause upgrade errors or cause staff to select the wrong package.

Retry the Upgrade With the Correct Image

  • After confirming the software image is official, digitally signed, unmodified, and in the correct location, retry the software upgrade.
  • Confirm the MAC 5 accepts the software image and allows the upgrade process to continue.
  • If the error clears, complete the upgrade per approved facility procedure.
  • Verify normal startup and return the unit to service only after functional checks are completed.

Do Not Attempt Workarounds

  • Do not attempt to bypass the digital signature requirement.
  • Do not rename files to force recognition.
  • Do not use software images from another MAC 5 unless confirmed appropriate by GE HealthCare.
  • Do not attempt internal repair for this error.
  • Keep troubleshooting limited to verifying the software package, media, file location, and approved source.

If the Problem Persists

If the error continues after verifying the USB drive or shared folder, file location, and software source, the software image should be considered invalid for this upgrade.

External causes such as incorrect media placement, wrong file location, incomplete copy, unsupported USB media, shared-folder access issue, and unapproved software source should be ruled out before escalation.

Remove the MAC 5 from service if the software upgrade is required before clinical use.

Label the device Out of Service if it cannot be returned safely to normal operation.

Send the device for repair or bench evaluation, and contact GE HealthCare Service Support to obtain the correct digitally signed host software image or confirm the correct upgrade package for the device.

Clinical Use Tip

Never troubleshoot software upgrade errors on a device being used for patient ECG acquisition.

Move the patient to a backup ECG machine first if testing is clinically needed.

Maintain therapy and testing continuity by using another verified ECG machine until the MAC 5 upgrade issue is resolved and the unit passes functional verification.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"GE MAC 5 displayed Host Software Image Is Not Digitally Signed during attempted host software upgrade. Upgrade would not continue."

Cause

What was observed during troubleshooting.

Example:
"Host software image failed digital signature validation. External media and file location checked; software package suspected to be unsigned, incorrect, corrupted, modified, or not from approved GE HealthCare source."

Resolution

What action was taken.

Example:
"Stopped upgrade attempt. Verified upgrade media and file placement. Removed device from service pending correct digitally signed host software image from GE HealthCare Service Support."

Helpful Details to Include (If Known)

  • Alarm behavior or exact error behavior, including whether the error appeared immediately or after file selection
  • Accessories swapped, including USB flash drive tested or replaced
  • Power behavior during the attempted software upgrade
  • Environmental factors, including USB use, shared network folder use, or network availability
  • Indicator lights, screen messages, software image file name, software version, and source of the software package
  • Final device status, including returned to service or removed from service

Final Thought

This error should be treated as a software integrity protection message. Clinical Engineering should verify the external media, file location, and software source, but should not attempt to bypass digital signature validation. If the correct signed image is not available, escalate to GE HealthCare Service Support and document the failure clearly.

That is successful troubleshooting.
