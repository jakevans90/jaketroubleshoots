Hillrom Centrella Smart+ Hospital Bed

Nurse Call Integration Failure

Asset Type

Hospital Bed

Manufacturer

Hillrom

Model

Centrella Smart+

What This Guide Helps With

This guide addresses situations where the Hillrom Centrella Smart+ bed fails to integrate properly with the nurse call system. Symptoms may include:

  • Nurse call alerts not triggering when bed exit occurs
  • Nurse call not acknowledging patient assistance requests
  • Intermittent or inconsistent signaling to the nurse call system

The focus is on external checks and system interface verification before assuming internal bed electronics failure.

Step-by-Step Troubleshooting

Verify Power to Bed

  • Ensure the bed is connected to a live AC outlet.
  • Check the bed’s main power indicator LED.
  • Why: Nurse call integration will not work if the bed has no power.

Check Bed Network/Communication Cables

  • Inspect cables connecting the bed to the nurse call interface (RJ45, serial, or proprietary connector).
  • Ensure cables are firmly seated, undamaged, and routed properly.
  • Why: Loose or damaged cables are a common cause of communication failure.

Verify Nurse Call System Status

  • Confirm the nurse call system is powered and operational.
  • Check for any active system alarms or faults that could prevent communication.
  • Why: Bed-side signals rely on a functioning nurse call system.

Check Bed Settings for Nurse Call Integration

  • On the bed’s control panel, navigate to the settings menu for nurse call or alert configuration.
  • Confirm integration is enabled and configured for the correct room or station.
  • Why: Incorrect configuration can prevent alerts from being sent.

Swap with a Known-Good Bed or Room

  • Move the bed to a room with a functioning nurse call system (if feasible) or replace it temporarily with a bed known to work.
  • Why: Isolates whether the issue is the bed or the nurse call interface.

Inspect for Software Updates or Bed Firmware Issues

  • Check Hillrom documentation for the latest firmware that addresses nurse call integration.
  • Why: Some integration issues are resolved via software updates.

Test Bed Functions Not Related to Nurse Call

  • Operate bed adjustments and bed exit sensors.
  • Why: Confirms that bed electronics and sensors are functional before escalating.

If the Problem Persists

External connections and settings have been verified. The issue may be an internal communication board failure or sensor interface fault.

  • Remove the bed from service.
  • Label the bed “Out of Service.”
  • Send to Hillrom service or in-house repair for bench evaluation.

Note: Stopping here prevents unnecessary risk and preserves patient safety.

Clinical Use Tip

Do not attempt nurse call troubleshooting while a patient is in bed if it could interrupt care. Move the patient to another bed if necessary before continuing testing. Only perform communication tests in a safe, unoccupied environment.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“Bed is not triggering nurse call alerts.”

Cause

What was observed during troubleshooting.

Example:
“Bed exit sensors functional; integration cables secure; nurse call system operational; bed settings confirmed.”

Resolution

What action was taken.

Example:
“Bed removed from service and sent for manufacturer evaluation.”

Helpful Details to Include

  • Outlet verified: Yes/No
  • Cables checked and swapped: Yes/No
  • Bed exit sensors functional: Yes/No
  • Firmware version: [Documented version]
  • Nurse call system status: Operational/Alarm

Final Thought

Patient safety is paramount—always confirm power, connections, and system status before assuming internal bed failure. Proper documentation ensures smooth repair and minimal patient disruption. Escalate appropriately and avoid unnecessary attempts to repair internal electronics on-site.

That is successful troubleshooting.
