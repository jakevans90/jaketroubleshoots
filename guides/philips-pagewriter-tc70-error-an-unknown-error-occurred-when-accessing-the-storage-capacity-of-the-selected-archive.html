Philips PageWriter TC70

Error: An unknown error occurred when accessing the storage capacity of the selected archive

Asset Type

Electrocardiograph (EKG) Machine

Manufacturer

Philips

Model

PageWriter TC70

What This Guide Helps With

This guide helps troubleshoot the archive storage capacity error on the Philips PageWriter TC70. This condition may prevent ECG storage, retrieval, export, or archive access when the cardiograph cannot determine the available capacity of the selected archive location.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • If the PageWriter TC70 is needed for active ECG acquisition and archive access is failing, do not rely on the affected archive function until the issue is resolved.
  • Use another verified ECG machine if ECG capture, storage, or transfer is required for patient care.
  • Expected outcome: Patient testing is not delayed or dependent on a device with an unresolved storage/archive error.

Verify the Reported Error and Behavior

  • Power on the PageWriter TC70 and attempt to access the same archive or storage destination that produced the error.
  • Confirm the exact message: “An unknown error occurred when accessing the storage capacity of the selected archive.”
  • Observe whether the error occurs during startup, when opening an archive, when saving an ECG, when exporting ECGs, when selecting removable media, or when accessing network storage or the internal archive.
  • Expected outcome: The error is confirmed and tied to a specific storage/archive action.

Identify the Selected Archive Location

  • Determine which archive the unit is trying to access.
  • Check whether the selected archive is internal storage, USB/removable media, a network archive, a shared folder or server location, a previous ECG archive location, or a misconfigured destination.
  • Expected outcome: The affected storage path is identified before hardware failure is assumed.

Check for Removable Media Issues

  • If the selected archive uses USB or removable storage, remove the media and inspect it.
  • Check for a damaged USB connector, wrong media inserted, full storage device, write-protected media, unsupported format, corrupted files, or a loose connection.
  • Try a known-good, approved USB storage device if allowed by facility policy.
  • If the error clears with different media, the original storage device is the likely cause and should not be returned to clinical use.

Check Internal Storage Availability

  • If the error occurs when using the internal archive, review available storage if the menu can still be accessed.
  • Look for inability to save ECGs, inability to retrieve previous ECGs, slow archive loading, repeated storage errors, system freezes, or corrupted or missing records.
  • Expected outcome: Internal archive function is either usable or clearly failing.

Restart the Cardiograph and Recheck

  • Power the PageWriter TC70 off normally.
  • Wait briefly, then power it back on and repeat the archive action.
  • Expected outcome: Temporary software or storage access faults clear after restart.
  • If the error does not return and the unit saves, retrieves, prints, and transfers ECGs normally, continue with functional verification before returning to service.

Check Power and Battery Condition

  • Connect the cardiograph directly to a known-good AC outlet.
  • Confirm the unit powers normally and does not show low battery, shutdown, or abnormal startup behavior.
  • Expected outcome: The storage/archive error is not being caused by unstable power or interrupted startup.

Verify Network Archive Availability

  • If the archive is network-based, confirm the network connection before assuming device failure.
  • Check the Ethernet cable connection, network jack status, wireless connection if used, correct network profile, server/archive availability, IP or network configuration, recent network changes, and whether other ECG machines can access the same archive.
  • If other devices also cannot reach the archive, escalate to IT or the ECG management system administrator.

Check Archive Configuration

  • Review the selected archive destination and configuration.
  • Confirm the correct archive is selected, the storage destination still exists, credentials or access rights have not changed, the folder path or server address is correct, and the device is not pointed to an old or unavailable archive.
  • Expected outcome: The archive configuration matches the intended facility workflow.

Test Basic ECG Functions

  • Run a controlled functional check.
  • Verify the PageWriter TC70 can power on normally, acquire an ECG, print an ECG, display patient data correctly, save an ECG if storage is available, retrieve a stored ECG if archive access is available, and transfer or export according to facility workflow.
  • Expected outcome: Basic ECG operation is confirmed, or the storage/archive failure is shown to affect clinical workflow.

Isolate the Failure

  • If the error occurs only with one USB device, suspect failed, full, corrupted, or unsupported removable media.
  • If the error occurs only with one network archive, suspect configuration, permissions, server access, or network availability.
  • If the error occurs with all archive locations, suspect cardiograph software, internal storage, or system-level failure.
  • If the error appears after restart and prevents ECG storage or retrieval, treat the device as unreliable for normal use.

Stop When Resolved

  • If the error clears after correcting media, network access, archive selection, or configuration, perform a final operational check.
  • Confirm the device can save, retrieve, print, and transfer ECGs according to facility workflow.
  • Return the PageWriter TC70 to service only after archive function is verified.

If the Problem Persists

If external storage, network access, archive configuration, power, and basic workflow issues have been ruled out, the persistent error may indicate an internal storage, software, or system-level failure.

Remove the PageWriter TC70 from service.

Label it Out of Service.

Send it for repair, software evaluation, or Philips support review. The manufacturer-directed action for this error is to contact the Philips Response Center.

Clinical Use Tip

Never troubleshoot archive or storage errors while the ECG machine is needed for active patient testing. Move the patient workflow to another verified ECG machine first if ECG storage, retrieval, or transfer cannot be trusted.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Philips PageWriter TC70 displayed ‘An unknown error occurred when accessing the storage capacity of the selected archive’ when attempting to access the selected ECG archive."

Cause

What was observed during troubleshooting.

Example:
"Selected archive storage capacity could not be verified. Removable media, network archive access, power, and archive configuration were checked during troubleshooting."

Resolution

What action was taken.

Example:
"Verified reported error, checked selected archive location, inspected applicable media and network access, restarted device, and tested ECG save/retrieve/print workflow. Device was removed from service and escalated for Philips support review when the archive error persisted."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Storage and archive errors on an ECG machine are not just convenience issues. They can affect patient record capture, retrieval, and transfer. Start with simple external causes such as media, network access, power, and configuration, then escalate when the error remains unresolved. Good documentation helps show whether the issue was corrected locally or properly removed from service for repair.

That is successful troubleshooting.
