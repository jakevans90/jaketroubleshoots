Cardinal Health Kangaroo ePump

Error: 2 Too Many Graphic Screens in Memory

Asset Type

Feeding Pump

Manufacturer

Cardinal Health

Model

Kangaroo ePump

What This Guide Helps With

Error: 2 Too Many Graphic Screens in Memory indicates an internal memory overload condition that can cause startup failures, freezing, or inability to operate the feeding pump normally.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Remove the pump from active patient use if it is not functioning properly
  • Transition feeding to a backup device if needed
  • Ensure nutrition delivery continues without interruption

2. Verify the Reported Error and Behavior

  • Power on the pump and confirm the exact error message
  • Observe if Error: 2 appears during startup or operation
  • Note if the screen freezes, reboots, or fails to load properly

3. Perform a Full Power Cycle (Most Common Fix)

  • Turn the pump OFF
  • Disconnect from AC power
  • Remove the battery if removable
  • Wait 30–60 seconds
  • Reinstall battery and reconnect power
  • Power ON the device

4. Check for Abnormal Startup Behavior

  • Observe for delayed startup or screen glitches
  • Watch for repeated reboot cycles
  • Attempt basic navigation if the interface loads

5. Isolate as Device-Level Issue

  • Confirm no external accessories are connected or influencing operation
  • Verify AC adapter and power source are stable
  • Confirm no change in behavior after external checks

If the Problem Persists

If Error: 2 continues after power cycling and external checks, internal memory or firmware corruption is likely.

  • Remove the pump from service
  • Label the device as Out of Service
  • Send for repair or board-level evaluation

Clinical Use Tip

Never troubleshoot on an active patient. Move the patient to a backup feeding pump before attempting any troubleshooting steps.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Pump displays Error: 2 during startup and intermittently fails to boot."

Cause

What was observed during troubleshooting.

Example:
"Internal memory fault suspected; error persisted after full power cycle."

Resolution

What action was taken.

Example:
"Removed from service and sent for firmware reflash or control board evaluation."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Error: 2 Too Many Graphic Screens in Memory indicates an internal system limitation rather than a user setup issue. Start with simple resets, verify consistent behavior, and escalate appropriately to protect patient safety and device reliability.

That is successful troubleshooting.
