Spacelabs Healthcare Xprezzon

Module Not Recognized or Parameter Missing

Asset Type

Patient Monitor

Manufacturer

Spacelabs Healthcare

Model

Xprezzon

What This Guide Helps With

Troubleshooting missing parameters or unrecognized modules caused by seating, cabling, configuration, accessory, or module communication issues.

Step-by-Step Troubleshooting

Ensure Patient Safety First

If the Xprezzon monitor is connected to an active patient and a required parameter is missing, do not rely on incomplete monitoring.

Move the patient to another verified monitor or provide alternate monitoring according to clinical policy before troubleshooting.

Expected outcome: The patient is monitored safely before any device checks begin.

Verify the Reported Missing Parameter

Confirm what clinical staff reported as missing, such as ECG, SpO2, NIBP, invasive pressure, temperature, CO2, or another parameter.

Check whether the parameter is missing from the screen, showing dashes, unavailable, or not appearing after module connection.

Expected outcome: The specific missing parameter and symptom are clearly identified.

Check Module Seating

Inspect the affected module and confirm it is fully inserted into the monitor or module housing.

Remove and reinsert the module firmly without forcing it.

Watch for module indicator lights, screen changes, or parameter tiles returning.

Expected outcome: If the issue was poor seating, the monitor recognizes the module and the parameter returns. If resolved, stop here.

Inspect the Module Bay or Connector Area

Check the module connection area for bent pins, debris, fluid residue, damaged plastic, or signs of impact.

Do not use sharp tools or force the connector.

Expected outcome: The module bay appears clean, dry, and physically intact.

Power Cycle the Monitor Safely

If the monitor is not connected to an active patient, power the Xprezzon monitor off, wait briefly, and power it back on.

Observe whether the monitor detects the module during startup.

Expected outcome: A temporary communication issue may clear after restart. If the module is recognized, verify normal operation and stop.

Try a Known-Good Module of the Same Type

Swap the suspect module with a known-good compatible Spacelabs module, if available.

Use the same monitor and module slot.

Expected outcome:

  • If the known-good module works, the original module may be faulty.
  • If the known-good module also fails, the issue may involve the monitor, module bay, configuration, or communication path.

Try the Suspect Module on Another Compatible Monitor

Install the suspect module on another known-good compatible Spacelabs monitor or module housing.

Expected outcome:

  • If the module works elsewhere, the original monitor or module bay should be evaluated.
  • If the module fails on another monitor, the module should be removed from service.

Check Patient Cables and Sensors

Confirm the correct patient cable, lead set, SpO2 sensor, temperature probe, pressure cable, or CO2 accessory is connected.

Inspect for damaged cable jackets, broken pins, loose connectors, fluid contamination, or incorrect accessory type.

Replace with known-good accessories when available.

Expected outcome: If the parameter was missing due to an accessory fault, the parameter should return after cable or sensor replacement.

Confirm the Parameter Is Enabled or Displayed

Review the monitor layout and parameter display settings to confirm the parameter is not hidden, minimized, or assigned to another screen area.

Avoid making clinical configuration changes unless authorized by facility policy.

Expected outcome: The parameter appears if it was hidden or not selected for display.

Check for Alarm or Message Indicators

Look for monitor messages such as module error, parameter unavailable, cable off, sensor off, lead fail, or communication error.

Document the exact wording.

Expected outcome: Any displayed message helps narrow whether the issue is module detection, accessory connection, or parameter configuration.

Inspect for Physical or Environmental Damage

Check the monitor and module for signs of drops, cracked housing, fluid ingress, corrosion, unusual heat, burning smell, or intermittent operation when gently handled.

Expected outcome: Any evidence of damage supports removal from service and repair evaluation.

Confirm Final Status

After reseating, swapping, restarting, and checking accessories, verify whether the missing parameter returned and remained stable.

If the module repeatedly drops out or the parameter disappears again, treat the issue as unresolved.

Expected outcome: The monitor either passes functional verification or is removed from service.

If the Problem Persists

If the module is still not recognized or the parameter remains missing after external checks, common causes such as poor seating, cable failure, accessory mismatch, display setting issues, and temporary communication faults have been ruled out.

The issue may involve the module, module bay, internal communication hardware, monitor configuration, or system electronics.

The device should be removed from service, labeled Out of Service, and sent for repair or bench evaluation.

Knowing when to stop is proper troubleshooting. Do not continue using a patient monitor that cannot reliably display required parameters.

Clinical Use Tip

Do not troubleshoot missing monitoring parameters while the patient depends on that monitor for active surveillance. Transfer the patient to another verified monitor first, especially if ECG, SpO2, NIBP, invasive pressure, or CO2 monitoring is clinically required.

Maintain therapy and monitoring continuity by using a backup monitor before continuing troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Nursing reported that the Spacelabs Xprezzon monitor would not recognize the parameter module and the SpO2 parameter was missing from the display."

Cause

What was observed during troubleshooting.

Example:
"Module was reseated and accessories were swapped, but the same module failed to be recognized on multiple compatible monitors."

Resolution

What action was taken.

Example:
"Removed the suspect module from service, labeled it Out of Service, and sent it for bench evaluation while the monitor was returned to use with a known-good module."

Helpful Details to Include (If Known)

  • Parameter missing from display
  • Module type affected
  • Module slot or bay used
  • Whether module was reseated
  • Whether a known-good module was tested
  • Whether the suspect module worked on another monitor
  • Cable or sensor swaps performed
  • Any monitor messages displayed
  • Indicator lights on the module
  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Evidence of fluid, impact, heat, or connector damage
  • Final device and module status

Final Thought

A missing parameter on a patient monitor should be treated as a patient safety issue first and a technical issue second. Start with module seating, accessories, display settings, and known-good swaps before suspecting internal failure. Clear CCR documentation helps identify whether the failure follows the module, the monitor, or the accessory path.

That is successful troubleshooting.
