Drager Fabius GS

Alarm: O2 SENSOR FAIL !

Asset Type

Anesthesia Machine

Manufacturer

Drager

Model

Fabius GS

What This Guide Helps With

O2 SENSOR FAIL ! alarm indicating inaccurate or absent oxygen concentration readings, which can lead to improper gas delivery and unsafe anesthesia conditions.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Immediately remove the device from active patient use if oxygen readings are unreliable
  • Transition patient to a backup anesthesia machine or ventilation source

2. Verify the Reported Alarm and Behavior

  • Confirm the O2 SENSOR FAIL ! alarm is displayed
  • Observe if O2 readings are missing, erratic, or fixed
  • Determine when the alarm occurs (startup vs during use)

3. Check O2 Sensor Connection

  • Ensure the sensor is properly installed in the housing
  • Verify the cable is securely connected
  • Inspect for loose, damaged, or disconnected wiring

4. Perform O2 Sensor Calibration

  • Initiate the calibration procedure per device workflow
  • Allow the sensor to equilibrate to room air if required
  • Complete calibration without interruption

5. Verify Sensor Condition (Wear/Expiration)

  • Check installation date if available
  • Inspect for physical damage or contamination
  • Confirm sensor is within usable life

6. Replace O2 Sensor if Needed

  • Replace the sensor capsule if calibration fails or sensor is expired
  • Perform calibration after replacement

7. Check O2 Sensor Housing / Module

  • Inspect the sensor housing unit for damage
  • Check internal connection points if accessible externally

8. Stop When Resolved

  • Verify stable and accurate O2 readings
  • Ensure alarm is cleared
  • Return device to service only after confirmation

If the Problem Persists

If external causes have been ruled out, including connections, calibration, and sensor replacement:

  • Remove the device from service
  • Label the device Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot on an active patient. Always move the patient to a backup device before addressing O2 sensor issues.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Unit displaying O2 SENSOR FAIL ! alarm with inaccurate oxygen readings during setup."

Cause

What was observed during troubleshooting.

Example:
"O2 sensor failed calibration and was found to be expired."

Resolution

What action was taken.

Example:
"Replaced O2 sensor, performed calibration successfully, verified accurate readings, and returned unit to service."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Oxygen monitoring is critical for safe anesthesia delivery. Always address calibration and connection issues first, but escalate quickly if accuracy cannot be restored.

That is successful troubleshooting.
