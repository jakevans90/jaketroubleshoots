STERIS STERIS AMSCO 4085

Error: DUAL COMMANDE

Asset Type

Surgical Table

Manufacturer

STERIS

Model

STERIS AMSCO 4085

What This Guide Helps With

This guide addresses the Error: DUAL COMMANDE condition, which occurs when both the foot pedal and hand control are used simultaneously, creating a control conflict where the system prioritizes hand control input.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Stop all table movement immediately
  • Ensure the patient is stable and secure
  • Do not continue positioning until control behavior is understood

Verify the Reported Error and Behavior

  • Attempt to operate the table using both controls
  • Check for “DUAL COMMANDE” message
  • Observe if only one control responds (typically hand control)

Check for Simultaneous Control Use

  • Ensure only one control is used at a time
  • Confirm with staff how controls are being used
  • Avoid pressing both controls simultaneously

Inspect Foot Pedal for Stuck or Continuous Input

  • Check for physical obstruction or debris
  • Ensure pedal returns to neutral position
  • Disconnect foot pedal and test hand control only

Inspect Hand Control Buttons

  • Check for stuck or continuously pressed buttons
  • Test each button individually
  • Inspect for damage or contamination

Test Controls Independently

  • Operate table using only the hand control
  • Operate table using only the foot pedal
  • Confirm normal operation with no error

Stop When Resolved

  • Return device to service once issue is corrected
  • Educate staff on proper control usage if needed

If the Problem Persists

If the error continues even when only one control is used, external causes have been ruled out. This may indicate an internal control or input circuit issue.

  • Remove the device from service
  • Label as Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot table controls while a patient is in a vulnerable surgical position. Move the patient to a backup device before performing troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Table displaying DUAL COMMANDE error during positioning; inconsistent response from controls."

Cause

What was observed during troubleshooting.

Example:
"Simultaneous input detected from foot pedal and hand control; foot pedal found partially engaged causing continuous signal."

Resolution

What action was taken.

Example:
"Freed and verified proper function of foot pedal; confirmed normal operation using single control input; returned to service."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

This error is often the result of normal system logic rather than a true failure. Understanding how control inputs interact can prevent unnecessary service calls and improve safe device operation.

That is successful troubleshooting.
