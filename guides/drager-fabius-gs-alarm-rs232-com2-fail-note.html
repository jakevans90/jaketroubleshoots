Drager Fabius GS

Alarm: RS232 COM2 FAIL !

Asset Type

Anesthesia Machine

Manufacturer

Drager

Model

Fabius GS

What This Guide Helps With

RS232 COM2 FAIL alarm indicating loss of communication with an external monitor or system connected to the COM2 port, resulting in missing data transmission or failure of monitoring integration.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • If external monitoring or data integration is required, confirm alternate monitoring is in place
  • Ensure patient data is still visible locally on the machine or backup system
  • Transition to alternate monitoring if needed

Verify the Reported Alarm and Behavior

  • Confirm the RS232 COM2 FAIL ! alarm is displayed
  • Observe when the alarm occurs (startup or during operation)
  • Check if external data is missing or frozen

Check if COM2 Is Actively Used

  • Determine if an external monitor or system is intended to be connected
  • Verify configuration with clinical staff
  • Confirm whether COM2 should be active

Inspect External Monitor Cable Connection

  • Ensure RS232 cable is securely connected at both ends
  • Reseat cable connections firmly
  • Inspect for bent pins or visible damage

Verify External Device Power and Function

  • Confirm the external device is powered on
  • Check for proper operation of the device
  • Restart the external device if needed

Swap or Test Cable

  • Replace with a known good RS232 cable if available
  • Observe if alarm clears after replacement

Check Configuration Settings

  • Verify correct port (COM2) is selected
  • Review communication settings if applicable
  • Disable COM2 if not in use

Reboot the System

  • Power cycle the anesthesia machine
  • Observe if communication resets and alarm clears

If the Problem Persists

If all external causes have been ruled out:

  • Remove the device from service
  • Label the unit as Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot communication alarms on an active patient. Always ensure monitoring continuity using backup systems before proceeding.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"RS232 COM2 FAIL alarm displayed and external monitor not receiving data."

Cause

What was observed during troubleshooting.

Example:
"Loose RS232 cable connection or COM2 port not configured for use."

Resolution

What action was taken.

Example:
"Reseated/replaced RS232 cable and verified external device operation. Adjusted settings or removed from service if unresolved."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Communication alarms like RS232 COM2 FAIL are often caused by external issues such as disconnected cables or unused ports. Always verify connections and configuration before escalating to hardware failure.
