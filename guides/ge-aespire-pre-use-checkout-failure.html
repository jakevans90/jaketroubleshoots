GE Healthcare Aespire

Pre-Use Checkout Failure

Asset Type

Anesthesia Machine

Manufacturer

GE Healthcare

Model

Aespire

What This Guide Helps With

Identifies causes of failed pre-use checks, including power, sensor, circuit, or connection issues before patient exposure.

Step-by-Step Troubleshooting

Ensure Patient Safety

  • Confirm no patient is connected to the machine.
  • Move the patient to an alternate anesthesia workstation if ventilation is required.

Check Power Source

  • Verify the machine is plugged into a functioning outlet.
  • Confirm the main power switch is ON.
  • Expected outcome: machine powers up without error.

Inspect Battery and UPS (if equipped)

  • Check battery status indicators and ensure sufficient charge.
  • Replace or recharge battery if low.
  • Expected outcome: pre-use checkout completes without power-related errors.

Examine Circuit and Gas Connections

  • Ensure breathing circuit, CO₂ absorber, and hoses are properly connected.
  • Verify vaporizers are seated correctly and locked.
  • Expected outcome: all sensors detect connections and readiness.

Check Sensors and Alarms

  • Inspect O₂, N₂O, and air sensors for proper function and no obstruction.
  • Check that pressure and flow sensors are unobstructed.
  • Expected outcome: sensor readings stabilize within normal ranges.

Verify Software and System Status

  • Confirm system self-test or software update has completed successfully.
  • Restart the machine if software appears frozen or unresponsive.
  • Expected outcome: pre-use checkout initiates correctly.

Run Pre-Use Checkout Again

  • Initiate the checkout procedure per standard workflow.
  • Observe for specific error codes or failure messages.
  • Expected outcome: successful completion of pre-use check.

If the Problem Persists

Common external causes have been ruled out.

Remove the device from service.

Label as Out of Service.

Send for GE Healthcare service evaluation or bench repair.

Knowing when to escalate prevents patient risk and unnecessary troubleshooting.

Clinical Use Tip

Never perform troubleshooting with an active patient attached.

Always move the patient to a functional machine first.

Document observations clearly for safe handoff to service personnel.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Pre-use checkout fails, error message displayed."

Cause

What was observed during troubleshooting.

Example:
"Power supply verified, connections secure, sensor checks complete; failure persists."

Resolution

What action was taken.

Example:
"Device removed from service, labeled Out of Service, sent to GE Healthcare for repair."

Helpful Details to Include (If Known)

  • Outlet tested and verified
  • Circuit connections checked
  • Sensor and alarm indicators observed
  • Error codes or messages noted
  • Machine restart attempted

Final Thought

Systematic troubleshooting protects patient safety and ensures efficient escalation. Accurate documentation supports service, minimizes downtime, and reinforces Clinical Engineering best practices.

That is successful troubleshooting.
