Baxter Novum IQ

Network Connectivity or EMR Sync Failure

Asset Type

Infusion Pump

Manufacturer

Baxter

Model

Novum IQ

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting situations where the Novum IQ infusion pump fails to connect to the hospital network or synchronize with the EMR/central system. Symptoms may include missing infusion documentation in the EMR, intermittent connectivity, network alarms, or failure to transmit infusion data. The guide emphasizes external checks, safe escalation, and patient safety rather than internal electronics repair.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Do not troubleshoot on a pump actively infusing a patient.
  • Move the patient to another verified pump before troubleshooting.
  • Only perform connectivity troubleshooting when patient safety is ensured.

Check Pump Power and Status

  • Ensure the pump is powered on and not in a “standby” or “error” state.
  • Confirm battery is adequately charged or connected to AC power.
  • Why: A powered-down pump cannot communicate with the network.

Verify Network Cable or Wi-Fi Connection

  • Inspect the network cable for damage and ensure it is securely connected.
  • If using Wi-Fi, confirm the pump is connected to the correct SSID.
  • Why: Loose or damaged cables or incorrect Wi-Fi credentials prevent network communication.

Confirm IP Address and Network Settings

  • Check the pump’s network configuration menu for a valid IP address, subnet mask, and gateway.
  • Ensure DHCP is functioning or the static IP does not conflict with another device.
  • Why: Network misconfiguration is a common cause of EMR sync failures.

Check Hospital Network Status

  • Verify that the network switch, access point, or port the pump is connected to is operational.
  • Confirm no recent network maintenance or firewall changes are blocking communication.
  • Why: External network issues can appear as pump connectivity problems.

Confirm EMR Integration Settings

  • Ensure the pump is registered in the EMR system with correct identifiers.
  • Verify that the integration server or middleware (e.g., Smart Pump Interface) is operational.
  • Why: If the server or integration settings are misconfigured, data will not sync.

Restart Pump and/or Network Connection

  • Reboot the pump and, if possible, reset the network connection.
  • Observe whether the pump successfully reconnects to the EMR.
  • Why: Some transient network errors are resolved with a reboot.

If the Problem Persists

All external causes (power, network cables, network switch, EMR registration) have been ruled out.

The issue is likely internal (network interface failure or software issue).

  • Remove the pump from service.
  • Label as “Out of Service.”
  • Send to Biomedical Engineering for bench evaluation or vendor repair.

Knowing when to escalate is proper troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“Pump not showing infusion data in EMR; network connection failure alarm present.”

Cause

What was observed during troubleshooting.

Example:
“Observed pump powered on, connected to network, valid IP, but EMR integration server unreachable.”

Resolution

What action was taken.

Example:
“Restarted pump and network connection; issue persisted. Pump removed from service and sent for bench evaluation.”

Helpful Details to Include

  • Network port tested: Yes/No
  • Wi-Fi credentials confirmed: Yes/No
  • IP address and subnet verified
  • Network switch or AP status
  • EMR integration server status
  • Pump reboot attempts
  • Final pump status (Out of Service / In Service)

Final Thought

Clinical troubleshooting relies on a logical approach: start with external checks, confirm network and EMR settings, and escalate when necessary. Ensuring patient safety and proper documentation protects both staff and patients while maintaining operational integrity.

That is successful troubleshooting.
