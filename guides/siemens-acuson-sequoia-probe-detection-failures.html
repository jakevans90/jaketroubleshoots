Siemens Healthineers ACUSON Sequoia

Probe Detection Failures

Asset Type

Ultrasound System

Manufacturer

Siemens Healthineers

Model

ACUSON Sequoia

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting situations where the ACUSON Sequoia ultrasound system fails to recognize or detect a connected probe. Common presentations include:

  • “No Probe Connected” or similar error messages
  • Intermittent probe recognition
  • System prevents imaging when the probe is connected
  • Error codes related to probe communication

This guide focuses on simple, external checks before considering internal repair.

Step-by-Step Troubleshooting

Verify Probe Connection

  • Ensure the probe is fully seated in the correct port.
  • Check for any dirt, debris, or corrosion on the probe connector or system port.
    Why: Loose or dirty connections are the most common cause of detection failure.

Inspect the Probe Cable

  • Look for visible damage such as cuts, kinks, or fraying.
  • Gently straighten any bends and ensure strain relief is intact.
    Why: Damaged cables can interrupt communication signals to the system.

Swap Ports (If Applicable)

  • If the system has multiple probe ports, try connecting the probe to a different port.
    Why: Determines if the issue is the probe or the system port.

Test with a Known-Good Probe

  • Connect a different, verified working probe to the same port.
    Why: Confirms whether the issue lies with the original probe or the system electronics.

Check System Software and Firmware

  • Verify that the system software supports the probe type being used.
  • Look for any recent updates or alerts from Siemens Healthineers regarding probe compatibility.
    Why: Software mismatches can cause detection errors even when hardware is functional.

Restart the System

  • Perform a controlled system shutdown and restart.
    Why: Resets system communication and can clear temporary faults.

Document Observations

  • Record probe type, port used, error codes, and any patterns noticed.
    Why: Documentation aids in future troubleshooting or escalation.

If the Problem Persists

External issues have likely been ruled out.

  • Remove the system or affected probe from service.
  • Label the probe/system Out of Service.
  • Send the probe/system to Siemens Healthineers service or internal bench evaluation.

Knowing when to stop is proper troubleshooting—do not attempt internal electronics repair.

Clinical Use Tip

  • Do not attempt to image a patient using a probe that is not recognized by the system.
  • Move the patient to a working system or verified probe before continuing the exam.
  • Always ensure that patient safety is prioritized while troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“No Probe Connected” message on the ACUSON Sequoia system.

Cause

What was observed during troubleshooting.

Example:
Probe cable or connector may be loose or damaged; system may not be recognizing the probe.

Resolution

What action was taken.

Example:
Verified connections and cable integrity, swapped probe port, tested with known-good probe, restarted system. If problem persisted, system and/or probe labeled Out of Service and sent for service evaluation.

Helpful Details to Include

  • Ports tested and results
  • Cables swapped
  • Any error codes displayed
  • Indicator lights on probe connector
  • Observed behavior after system restart
  • Final device and probe status

Final Thought

Systematic, stepwise troubleshooting ensures patient safety while preserving equipment integrity. By checking external connections first, verifying software, and testing known-good probes, Clinical Engineering can identify whether the issue is user-serviceable or requires escalation. Proper documentation using CCR reinforces accountability and efficiency.

That is successful troubleshooting.
