Jaeger Medical Vmax Encore

Network / EMR Communication Failure

Asset Type

Pulmonary Function Tester

Manufacturer

Jaeger Medical

Model

Vmax Encore

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting situations where a Jaeger Medical Vmax Encore pulmonary function testing system cannot communicate with the hospital network or Electronic Medical Record (EMR). Symptoms may include test results failing to upload, network connection errors, or inability to access patient data. The guide emphasizes verifying external connectivity and software settings before escalating to internal repairs.

Step-by-Step Troubleshooting

Confirm Device Power and Status

  • Ensure the Vmax Encore is powered on and fully booted.
  • Check indicator lights on both the PC and the device.

Why: Devices that are not fully powered or are frozen cannot establish network connections.

Verify Network Connectivity

  • Check Ethernet cables or Wi-Fi connection; ensure secure connection to hospital network.
  • Confirm the PC or workstation running Encore can access the network and other hospital systems.

Why: Network failures are the most common cause of EMR communication issues.

Check IP and Network Settings

  • Ensure the Vmax Encore or connected PC has a valid IP address, subnet mask, and gateway.
  • Confirm DNS servers are reachable.

Why: Incorrect IP or DNS settings prevent communication with EMR servers.

Test EMR Server Reachability

  • Ping the EMR server from the Vmax Encore PC.
  • If ping fails, document network error and escalate to IT/network team.

Why: Confirms whether the problem is local or on the server side.

Review Software Configuration

  • Open the Encore software and verify EMR integration settings: server address, port, and login credentials.
  • Ensure the software version is compatible with EMR interface.

Why: Misconfigured software settings can block data transmission even when network is functional.

Restart Software / Device

  • Close Encore software and reopen.
  • If needed, reboot the Vmax Encore PC.

Why: Software glitches can temporarily prevent communication.

Check Firewall / Security Software

  • Ensure the firewall or antivirus is not blocking Encore software or required ports.

Why: Security restrictions can prevent EMR communication.

If the Problem Persists

External network issues and software misconfigurations have been ruled out.

Device may have internal communication board or interface failure.

Next Steps:

  • Remove the Vmax Encore from service.
  • Label as Out of Service.
  • Send to the manufacturer or Clinical Engineering bench for repair.

Knowing when to stop is part of proper troubleshooting—do not attempt internal repairs beyond your scope.

Clinical Use Tip

  • Do not attempt to send patient results manually while troubleshooting the device.
  • Move the patient to another PFT system if available to continue testing safely.
  • Ensure patient safety and data integrity before and during troubleshooting.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“Test results are not uploading to EMR; network connection error displayed on Vmax Encore.”

Cause

What was observed during troubleshooting.

Example:
“Ethernet cable disconnected and EMR server unreachable from device; software misconfiguration verified.”

Resolution

What action was taken.

Example:
“Connected network cable, verified IP settings, rebooted PC, confirmed EMR connection restored.”

Helpful Details to Include

  • Indicator lights: Check PC network LEDs.
  • Ethernet/Wi-Fi connection status.
  • Ping or trace route results to EMR server.
  • Firewall or antivirus alerts during connection attempts.
  • Software version and EMR integration settings.

Final Thought

Clinical Engineering troubleshooting is most effective when it follows logical steps from simple external checks to more complex issues. Always verify patient safety first, escalate appropriately, and document actions clearly. Proper documentation and knowing when to stop protect both staff and equipment.

That is successful troubleshooting.
