Puritan Bennett 840

Exhalation Module Replacement and Errors

Asset Type

Ventilator

Manufacturer

Puritan Bennett

Model

840

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting and replacing the exhalation module on Puritan Bennett 840 ventilators. Exhalation module errors can cause abnormal ventilation, alarms, or patient safety risks. Many issues are external or mechanical, such as incorrect installation, contamination, or module wear. This guide focuses on step-by-step checks and proper escalation without internal board-level repair.

Step-by-Step Troubleshooting

Verify Patient Safety First

  • Ensure the ventilator is not actively supporting a patient before troubleshooting.
  • If connected, switch the patient to a backup ventilator.

Power and Alarm Status Check

  • Confirm the ventilator is powered on and no general system errors exist.
  • Note any specific exhalation module alarms or codes.

Inspect the Exhalation Module

  • Check for visible contamination, damage, or obstruction.
  • Ensure module is fully seated and locked in place according to the manufacturer’s guidance.

Disconnect and Re-seat Module

  • Power down ventilator if safe.
  • Remove the exhalation module carefully and inspect connectors.
  • Reinsert module, ensuring correct alignment and secure connection.

Run Ventilator Self-Test/Calibration

  • Power on ventilator.
  • Initiate the self-test or calibration sequence.
  • Observe if module errors persist.

Check Related Components

  • Inspect patient circuit and expiratory limb for kinks or obstructions.
  • Ensure filters are clean and properly seated.

Replace Module if Necessary

  • If errors persist, install a verified, clean, and compatible exhalation module.
  • Re-run ventilator self-test to confirm resolution.

If the Problem Persists

If after replacement the error remains, the ventilator should be:

  • Removed from service
  • Labeled Out of Service
  • Sent to the manufacturer or qualified bench repair team

Knowing when to stop is essential to prevent further damage or patient risk.

Clinical Use Tip

  • Never troubleshoot the exhalation module on a patient actively ventilated.
  • Always have a backup ventilator ready.
  • Verify alarm resolution before returning device to patient use.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“Exhalation module errors displayed; ventilator alarms.”

Cause

What was observed during troubleshooting.

Example:
“Module misaligned and/or contaminated; self-test failed.”

Resolution

What action was taken.

Example:
“Module removed, cleaned, and re-seated; self-test run and passed. Replaced module when errors persisted; device verified operational.”

Helpful Details to Include

  • Ventilator powered and outlet verified
  • Module seated and clean
  • Self-test results recorded
  • Filters and circuit inspected
  • Alarm codes documented

Final Thought

Stepwise troubleshooting ensures patient safety, minimizes unnecessary replacements, and supports proper documentation. Correctly replacing the exhalation module restores ventilator function without risking internal damage or patient harm.

That is successful troubleshooting.
