Drager Primus IE

Agent Delivery or Vaporizer Recognition Errors

Asset Type

Anesthesia Machine

Manufacturer

Drager

Model

Primus IE

What This Guide Helps With

Identifies causes for vaporizer not recognized, agent delivery failure, or inaccurate readings before patient exposure.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Confirm no patient is connected to the device. Move the patient to an alternate anesthesia machine if needed.

Verify Machine Power

  • Ensure the anesthesia machine is powered on and running its initial self-test without error. A stable power source prevents false vaporizer errors.

Check Vaporizer Installation

  • Ensure the vaporizer is correctly seated in its mounting slot.
  • Confirm the locking mechanism engages fully.
  • Expected Outcome: Machine should detect the vaporizer. Misalignment often prevents recognition.

Inspect Vaporizer Type and Compatibility

  • Verify the vaporizer matches the Primus IE’s approved agent list.
  • Cross-check model and fill capacity markings.
  • Expected Outcome: Only compatible vaporizers are recognized; incompatible types can trigger errors.

Check Vaporizer Filling and Level Sensor

  • Ensure the agent level is sufficient.
  • Look for leaks or spilled agent around the connection port.
  • Expected Outcome: Low or empty vaporizer can cause the machine to fail recognition.

Examine Sensor Connections

  • Inspect electrical or mechanical connectors on the vaporizer and machine for dirt, debris, or corrosion.
  • Clean connectors carefully if needed.
  • Expected Outcome: Clean, secure connectors restore proper communication.

Reset the Machine

  • Power down, wait 30 seconds, then power up.
  • Re-run the pre-use check or self-test.
  • Expected Outcome: Temporary communication faults may clear.

Swap with a Known Good Vaporizer

  • Replace the suspected vaporizer with a verified working one.
  • Expected Outcome: If recognition succeeds, the original vaporizer may be defective.

If the Problem Persists

External checks have been ruled out.
Remove the machine from clinical service.
Label as Out of Service.
Send to manufacturer-authorized service or internal bench evaluation.
Note: Continuing to use a machine with unrecognized vaporizer risks incorrect agent delivery.

Clinical Use Tip

Never attempt troubleshooting on an active patient.
Move the patient to a backup anesthesia machine first.
Ensure vaporizers are only installed and removed when the machine is powered off.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Primus IE fails to recognize sevoflurane vaporizer during pre-use check."

Cause

What was observed during troubleshooting.

Example:
"Vaporizer misaligned and connector pins partially obstructed."

Resolution

What action was taken.

Example:
"Corrected seating, cleaned connectors, pre-use check completed successfully."

Helpful Details to Include (If Known)

  • Machine self-test status and error codes
  • Vaporizer type and fill level
  • Any unusual clicks, resistance, or odors during installation
  • Connector condition
  • Final machine status after troubleshooting

Final Thought

Patient safety is paramount when agent delivery fails. A methodical approach from basic checks to escalation ensures no compromise occurs. Accurate documentation supports clinical confidence and ongoing equipment reliability.

That is successful troubleshooting.
