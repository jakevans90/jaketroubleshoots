Drager Primus IE

Display or Drager SmartPilot Errors

Asset Type

Anesthesia Machine

Manufacturer

Drager

Model

Primus IE

What This Guide Helps With

Identifies causes for display malfunctions, touchscreen errors, or SmartPilot integration issues before patient exposure or internal troubleshooting.

Step-by-Step Troubleshooting

1. Ensure Patient Safety

  • Confirm no patient is connected to the machine.
  • Move the patient to another anesthesia workstation if ventilation is required.

2. Check Power and Connections

  • Verify the machine is connected to a stable power source.
  • Inspect power cords for damage or loose connections.
  • Confirm battery (if present) is charged.
  • Expected outcome: Device powers on with normal startup sequence.

3. Inspect Display and Interface

  • Check for visible cracks, unresponsive touchscreen areas, or flickering.
  • Clean screen gently with manufacturer-approved wipes.
  • Expected outcome: Display responds to touch or button input.

4. Verify SmartPilot Integration

  • Confirm SmartPilot module is correctly seated in the connector.
  • Check communication cables between the anesthesia machine and SmartPilot.
  • Expected outcome: Device recognizes SmartPilot module; no communication errors.

5. Restart and Soft Reset

  • Power down the machine completely, wait 30 seconds, and restart.
  • Expected outcome: Display initializes normally; SmartPilot errors clear if caused by software glitch.

6. Check for Software Alerts or Updates

  • Review system messages or error codes on startup.
  • If recent updates were applied, confirm installation completed successfully.
  • Expected outcome: No persistent error messages; system operates normally.

If the Problem Persists

External checks have been ruled out; the issue may be internal software or display hardware failure.

Remove the device from service.

Label Out of Service.

Send for repair or bench evaluation.

Stopping here ensures patient safety and proper escalation.

Clinical Use Tip

Never troubleshoot display or SmartPilot errors with a patient attached.

Always confirm another anesthesia machine is available to safely continue patient care.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Display intermittently unresponsive; SmartPilot module not recognized."

Cause

What was observed during troubleshooting.

Example:
"Module connector seating checked; touchscreen and cables visually normal; errors persist after power cycle."

Resolution

What action was taken.

Example:
"Machine removed from service, labeled Out of Service, sent for repair/bench evaluation."

Helpful Details to Include (If Known)

  • Startup error codes displayed
  • Touchscreen responsiveness
  • Module seating and cable condition
  • Power source and battery status
  • Any recent software updates applied
  • Final device status: OOS

Final Thought

Patient safety comes first—never rely on a malfunctioning display or SmartPilot module. A logical approach ensures external causes are ruled out before repair escalation, preserving both device function and clinical workflow.

That is successful troubleshooting.
