Philips IntelliVue MP Series

Alarm: CHECK INPUT DEVICES

Asset Type

Patient Monitor

Manufacturer

Philips

Model

IntelliVue MP Series

What This Guide Helps With

This guide helps troubleshoot the CHECK INPUT DEVICES alarm on a Philips IntelliVue MP Series patient monitor when the monitor detects a defective Navigation Point, front panel control, touchscreen input issue, or connected input device failure.

This condition can affect the ability to navigate menus, acknowledge alarms, change settings, or operate the monitor reliably during patient care.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Confirm the Philips IntelliVue MP Series monitor is not the only active source of patient monitoring before troubleshooting input controls.
  • If the monitor cannot be operated normally, alarms cannot be acknowledged, or patient settings cannot be accessed, have clinical staff transfer monitoring to another working monitor.
  • Expected: Patient monitoring remains safe while the input device issue is evaluated.

Verify the Reported Alarm and Behavior

  • Confirm the monitor is displaying CHECK INPUT DEVICES.
  • Observe whether the INOP appears during startup, after pressing or turning the Navigation Point, when using a touchscreen, keypad, mouse, remote SpeedPoint, or another input device.
  • Check whether the monitor still responds to user input.
  • Expected: The CHECK INPUT DEVICES message is confirmed and associated with a specific input device or control behavior.

Remove the Monitor From Active Patient Use if Controls Are Unreliable

  • If the monitor cannot be navigated reliably, remove it from patient use.
  • Do not leave a monitor in service if staff cannot access menus, silence alarms, change settings, or confirm displayed information.
  • Expected: The device is not used for active patient care until input function is confirmed.

Inspect the Navigation Point / Main Control Knob

  • Visually inspect the Navigation Point or main rotary control for sticking, cracks, liquid contamination, loose movement, failure to press inward, failure to rotate smoothly, or physical damage from impact.
  • Rotate and press the control several times while observing monitor response.
  • Expected: The Navigation Point moves normally and produces consistent on-screen response.

Check for Stuck or Damaged Front Panel Buttons

  • Inspect all physical buttons on the monitor front panel.
  • Look for buttons that are jammed, sunken, sticky, physically cracked, not responding, or triggering without being pressed.
  • Expected: All buttons move freely and respond only when pressed.

Check for Liquid or Cleaning Residue

  • Inspect around the Navigation Point, keys, touchscreen edge, and input connectors for signs of fluid ingress or dried cleaning residue.
  • If contamination is present, remove the monitor from service and follow facility cleaning and evaluation procedures.
  • Expected: No fluid, residue, or contamination is affecting input controls.

Disconnect External Input Devices

  • If connected, disconnect external input accessories one at a time, such as a mouse, keyboard, remote control device, external SpeedPoint, remote input device, or other connected control accessory.
  • Restart the monitor after disconnecting suspect accessories if appropriate and only when the monitor is off patient use.
  • Expected: The CHECK INPUT DEVICES message clears when the defective external input device is removed.

Inspect Input Device Cables and Connectors

  • Check connected input device cables for bent pins, loose connectors, damaged cable jackets, strain at the connector, or evidence of being pulled, crushed, or contaminated.
  • Reseat connections where externally accessible.
  • Expected: Cables and connectors are intact, secure, and free of damage.

Test With Known-Good Input Accessories

  • If the monitor uses removable external input devices, test with known-good compatible accessories.
  • Do not use questionable or non-compatible devices for confirmation.
  • Expected: The monitor operates normally with known-good input hardware.

Check Touchscreen or Display Input Behavior if Applicable

  • If the model or configuration includes touchscreen operation, verify whether touch input works consistently.
  • Look for no response to touch, random selections, delayed response, touch working in some areas but not others, or input occurring without contact.
  • Expected: Touch input is accurate and stable.

Power Cycle the Monitor Off Patient Use

  • Once the monitor is safely off patient use, perform a controlled power cycle.
  • Restart the monitor and observe whether CHECK INPUT DEVICES returns immediately or only after operating a specific control.
  • Expected: The INOP either clears after restart or returns in a reproducible way.

Isolate the Failure

  • Determine whether the issue follows a specific external input accessory, cable, connector, Navigation Point, touchscreen, front panel, or the monitor itself.
  • Expected: The fault is narrowed to an accessory, external control device, or internal monitor input hardware.

Stop When Resolved

  • If removing or replacing a defective external input device clears the CHECK INPUT DEVICES message, verify normal monitor operation.
  • Confirm no CHECK INPUT DEVICES INOP remains, navigation works correctly, buttons respond normally, alarms can be acknowledged, and settings and menus can be accessed.
  • If all checks pass, document the repair and return the monitor to service.

If the Problem Persists

If CHECK INPUT DEVICES remains after visual inspection, accessory removal, known-good input device testing, cable and connector checks, contamination inspection, and power cycling, external and common causes have been ruled out.

A persistent CHECK INPUT DEVICES condition strongly indicates a defective Navigation Point, front panel input assembly, touchscreen input interface, or related monitor hardware.

The monitor should be removed from service, labeled Out of Service, and sent for repair or bench evaluation.

Do not continue troubleshooting on an active patient monitor with unreliable controls.

Clinical Use Tip

Never troubleshoot input control problems on an active patient monitor if staff cannot reliably operate the device. Move the patient to a backup monitor first, maintain therapy and monitoring continuity, and only troubleshoot once safe alternate monitoring is in place.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Philips IntelliVue MP Series monitor displaying CHECK INPUT DEVICES INOP. User reported Navigation Point or input controls not responding consistently during operation."

Cause

What was observed during troubleshooting.

Example:
"Visual and functional checks performed. External input accessories and connectors inspected. CHECK INPUT DEVICES appears related to defective input device or Navigation Point after common external causes were ruled out."

Resolution

What action was taken.

Example:
"Removed monitor from patient use. Tested input controls, checked for stuck buttons and contamination, disconnected external input devices, and power cycled unit. Defective input device suspected. Monitor labeled Out of Service and sent for repair or bench evaluation."

Helpful Details to Include (If Known)

  • Alarm behavior, including when CHECK INPUT DEVICES appears and whether it occurs at startup or during use.
  • Accessories swapped, including any mouse, keyboard, remote SpeedPoint, or external input device tested.
  • Power behavior, including whether a power cycle cleared the INOP or the message returned immediately.
  • Environmental factors, including liquid exposure, cleaning residue, impact damage, or contamination near controls.
  • Indicator lights or display behavior, including touchscreen response, random input, delayed response, or front panel button issues.
  • Final device status, including whether the monitor was returned to service, labeled Out of Service, or sent for repair or bench evaluation.

Final Thought

CHECK INPUT DEVICES should be treated as a control reliability issue, not just a nuisance message. Clinical Engineering should verify patient safety, check simple external causes first, isolate the defective input device when possible, and escalate appropriately when the alarm persists. Clear CCR documentation helps support safe return to service or repair decisions.

That is successful troubleshooting.
