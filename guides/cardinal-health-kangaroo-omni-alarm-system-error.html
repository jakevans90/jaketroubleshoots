Cardinal Health Kangaroo OMNI

Alarm: System Error

Asset Type

Feeding Pump

Manufacturer

Cardinal Health

Model

Kangaroo OMNI

What This Guide Helps With

This guide helps troubleshoot Kangaroo OMNI System Error alarms that may stop pump operation, interrupt feeding or flushing, or indicate a device-level system fault.

A System Error should be treated differently from routine feeding set, cassette, or battery alarms because it may involve a broader pump operating issue.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • If the Kangaroo OMNI is actively feeding or flushing and a System Error occurs, do not continue relying on the pump.
  • Stop use of the affected pump and ensure the patient’s feeding or flushing therapy is managed by clinical staff using an appropriate backup pump if needed.
  • Expected outcome: Patient therapy is not dependent on a pump displaying a system-level error.
  • Why it matters: A System Error may affect pump operation and should not be ignored during patient care.

Verify the Reported Alarm and Behavior

  • Look at the pump display and confirm that the alarm shown is System Error.
  • Note when the alarm occurs, such as startup, during feeding, during flushing, after cassette loading, or intermittently during normal operation.
  • Check for associated symptoms such as stopped delivery, failure to restart, unexpected power-off, repeated alarms after restart, or a three-number code on the screen.
  • Expected outcome: The System Error condition is confirmed and the timing is documented.
  • Why it matters: This separates a true system-reported error from setup issues, user input delays, cassette alarms, feed errors, or battery alarms.

Follow the On-Screen Restart Instruction

  • Follow the restart instruction shown on the Kangaroo OMNI display.
  • Power the pump off as directed, then turn it back on.
  • Expected outcome: The pump restarts normally and the System Error clears.
  • Why it matters: A System Error may be temporary, and the manufacturer-directed first response is to restart the pump.

Confirm Whether the Alarm Returns

  • After restart, observe the pump through normal startup.
  • If clinically appropriate, test with a proper feeding set and a non-patient test setup.
  • Check whether the System Error returns during normal operation.
  • If the alarm does not return, continue with functional verification before returning the pump to service.
  • Expected outcome: The pump completes startup without returning to the System Error alarm.

Check External and Simple Causes First

  • Inspect the pump exterior for visible damage, fluid intrusion, contamination, cracked housing, damaged buttons, or signs of impact.
  • Confirm the pump is physically intact, clean, and dry.
  • Why it matters: Physical damage or fluid contamination can contribute to system-level faults.

Check Power Source and Adapter Connection

  • Connect the Kangaroo OMNI to a known-good wall outlet using the correct power adapter.
  • Confirm the adapter is fully inserted into the pump power port.
  • Avoid relying on battery power during troubleshooting.
  • Expected outcome: Pump powers normally from AC power and does not lose power during startup.
  • Why it matters: Unstable power can mimic or contribute to abnormal pump behavior.

Remove Accessories and Reload Only What Is Needed

  • Remove the feeding set or cassette if installed.
  • Inspect for obvious cassette misloading, damage, fluid leakage, or contamination around the cassette loading area.
  • Reload a clean, known-good Kangaroo OMNI feeding set if testing is needed.
  • Expected outcome: The pump recognizes the set normally and does not immediately return to System Error.
  • Why it matters: Although System Error is not usually an accessory-specific alarm, external setup problems should still be ruled out before assuming internal failure.

Check for Repeatability

  • Power cycle the pump again and observe whether the System Error returns.
  • Record the exact alarm name, the three-number code displayed, when it appears, whether it occurs on AC power, and whether it occurs with or without a feeding set installed.
  • Expected outcome: The failure pattern is clear enough for repair escalation or manufacturer support.
  • Why it matters: The three-number code helps identify the system fault category and should be documented before the pump is powered off.

Stop When Resolved

  • If the System Error clears after restart and does not return during functional verification, document the finding.
  • Return the pump to service only after confirming normal operation.
  • Include that the alarm was not reproducible after restart and that external checks passed.
  • Expected outcome: Pump operates normally without repeating the System Error.

If the Problem Persists

If the System Error returns after turning the pump off and back on, external and common causes have been ruled out.

Document the three-number code shown on the screen, press Power Off, remove the pump from service, and label it Out of Service.

A persistent System Error strongly indicates an internal system fault or device-level failure requiring repair, bench evaluation, or manufacturer support.

Do not continue repeated restarts in patient care areas. Knowing when to stop is proper troubleshooting.

Clinical Use Tip

Never troubleshoot a System Error on an active patient setup. Move the patient to a backup device first before continuing. A pump with an unresolved System Error should not be trusted for patient use.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Kangaroo OMNI feeding pump displayed System Error during use. Pump stopped normal operation and required restart. Three-number code was present on screen."

Cause

What was observed during troubleshooting.

Example:
"System Error persisted after power cycle. AC power, external condition, cassette area, and feeding set setup were checked. Findings indicate likely internal device-level system fault."

Resolution

What action was taken.

Example:
"Recorded three-number System Error code, powered pump off, removed device from patient use, labeled Out of Service, and sent for repair or manufacturer support evaluation."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

A Kangaroo OMNI System Error should be approached as a device-reported fault, not a routine feeding setup alarm. Start with patient safety, confirm the exact alarm, perform the directed restart, check simple external causes, and document the displayed code if the error returns. Escalating a persistent System Error is the correct Clinical Engineering decision.

That is successful troubleshooting.
