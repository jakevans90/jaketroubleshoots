ICU Medical Colleague

Drug Library Not Loading

Asset Type

Infusion Pump

Manufacturer

ICU Medical

Model

Colleague

What This Guide Helps With

This guide addresses situations where the ICU Medical Colleague infusion pump fails to load its drug library. The focus is on verifying external factors such as power, connectivity, and data access before assuming internal software or hardware failure.

Step-by-Step Troubleshooting

Verify Power Source

  • Ensure the pump is connected to a functioning AC outlet or fully charged battery.
  • Check for blown fuses or tripped circuit breakers.
  • Why: Insufficient or unstable power can prevent the pump from loading software modules.

Check Battery Status

  • If running on battery, confirm it has adequate charge. Swap with a known-good battery if available.
  • Why: Some pumps restrict access to non-critical functions, including drug libraries, on low battery.

Inspect Drug Library Media

  • If the pump uses a USB, SD card, or other media for the drug library, ensure it is properly inserted.
  • Remove and reseat the media to ensure proper connection.
  • Why: Corrupt or improperly seated media can prevent the library from loading.

Check Software Updates

  • Confirm the pump’s software is up to date. Look for recent patches or updates applied incorrectly.
  • Why: Outdated or partially installed software can block drug library access.

Verify Connectivity (if network-based library)

  • Ensure the pump is connected to the hospital network or server hosting the drug library.
  • Check network cables or Wi-Fi signal.
  • Why: If the library is stored remotely, loss of connection can prevent loading.

Observe Error Messages and Indicator Lights

  • Note any error codes or alerts displayed when attempting to load the library.
  • Record indicator lights and audible alarms.
  • Why: Specific error codes can help narrow down the issue for repair.

Restart the Pump

  • Power cycle the device after checking the above items.
  • Attempt to reload the drug library.
  • Why: A restart can clear temporary software glitches.

If the Problem Persists

All external causes (power, media, battery, connectivity) have been ruled out.

The issue is likely an internal software or hardware fault.

Action:

  • Remove the device from service.
  • Label as Out of Service.
  • Send to Clinical Engineering bench for software reload or manufacturer repair.
  • Knowing when to escalate is proper troubleshooting.

Clinical Use Tip

  • Do not attempt to load the drug library on a patient actively receiving medication.
  • Move the patient to another functioning pump first.
  • Ensure all troubleshooting is performed in a safe, offline environment.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Pump will not load the drug library; error code displayed on startup."

Cause

What was observed during troubleshooting.

Example:
"Power source and media verified; network connection confirmed; error persists after restart."

Resolution

What action was taken.

Example:
"Device removed from service, labeled Out of Service, and sent for bench evaluation."

Helpful Details to Include

  • AC outlet tested
  • Battery swapped
  • Drug library media reseated
  • Error codes recorded: [Insert code]
  • Pump restart performed
  • Final status: Out of Service

Final Thought

Troubleshooting infusion pumps requires a patient-safety-first mindset. Start with the simplest external checks and escalate only when necessary. Proper documentation ensures continuity of care and accountability for device maintenance.

That is successful troubleshooting.
