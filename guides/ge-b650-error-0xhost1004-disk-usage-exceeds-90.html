GE Healthcare CARESCAPE B650

Error: 0xHOST1004 Disk usage exceeds 90%

Asset Type

Patient Monitor

Manufacturer

GE Healthcare

Model

CARESCAPE B650

What This Guide Helps With

Error 0xHOST1004 indicating disk usage over 90%, causing slow performance, data storage issues, or system instability.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Remove the monitor from patient use if performance is degraded or data is not saving properly
  • Transfer monitoring to a backup device
  • Confirm continuous patient monitoring on alternate equipment

2. Verify the Reported Error and Behavior

  • Power on the monitor and confirm Error: 0xHOST1004 is displayed
  • Observe when the error occurs (startup, during use, intermittent)
  • Check for lag, slow response, or inability to store data

3. Check for Excess Stored Data

  • Review stored patient cases, trends, and logs (if accessible)
  • Clear non-essential data per facility policy
  • Confirm available storage increases

4. Perform a Full Power Cycle

  • Power OFF the monitor
  • Disconnect AC power
  • Wait 30–60 seconds
  • Reconnect power and restart

5. Back Up Clinical and System Data

  • Access the Webmin interface
  • Back up clinical and platform settings
  • Print or save the licensing information page

6. Evaluate for Recurrence

  • Monitor system after reboot and cleanup
  • Check if disk usage warning returns
  • Determine if issue was temporary or persistent

7. Reinstall System Software (If Needed)

  • Proceed with approved software reinstallation process
  • Verify disk usage returns to normal levels

If the Problem Persists

If Error 0xHOST1004 continues after data cleanup and reboot, external causes have been ruled out and internal storage or software integrity may be compromised.

  • Remove the device from service
  • Label as Out of Service
  • Send for repair or full software reinstallation

Clinical Use Tip

Never troubleshoot on an active patient. Move the patient to a backup monitor before addressing storage or performance issues.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Monitor displaying Error 0xHOST1004 with slow response and inability to save patient data."

Cause

What was observed during troubleshooting.

Example:
"Disk usage exceeded 90% due to stored patient data and system logs."

Resolution

What action was taken.

Example:
"Backed up system via Webmin, cleared stored data, and performed power cycle. Unit escalated for software reinstallation if needed."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Storage-related errors are often preventable with routine data management. Always address external and user-level causes first, preserve system data, and escalate appropriately when internal limits are reached.

That is successful troubleshooting.
