Cardinal Health Kangaroo ePump

Error: 52 Graphic display timeout (write bytes failure)

Asset Type

Feeding Pump

Manufacturer

Cardinal Health

Model

Kangaroo ePump

What This Guide Helps With

This guide addresses Error 52 Graphic display timeout (write bytes failure), which indicates a communication failure between the control system and the display. This can result in a frozen screen, missing graphics, or an unresponsive pump, preventing safe operation and monitoring of enteral feeding.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Stop the feeding pump if actively infusing
  • Transition the patient to a backup feeding pump or method
  • Verify continued safe nutrition delivery

2. Verify the Reported Error and Behavior

  • Power on the pump and observe startup
  • Confirm Error 52 is displayed
  • Note if the screen is blank, frozen, or partially displaying
  • Determine if the error occurs at startup or during operation

3. Perform a Full Power Cycle

  • Turn off the pump
  • Disconnect from AC power
  • Remove the battery if accessible
  • Wait 30–60 seconds
  • Reconnect power and reinstall battery
  • Power the unit back on

4. Check Power Source Stability

  • Plug the device into a known-good outlet
  • Verify battery is properly seated
  • Observe for flickering or intermittent display behavior

5. Inspect for Physical Damage or Contamination

  • Check the display for cracks or discoloration
  • Inspect for fluid intrusion near the screen or keypad
  • Examine housing for signs of impact damage

6. Attempt Basic Function Check

  • Try navigating menus or starting a test setup
  • Observe if buttons respond despite display issues
  • Determine if the issue is display-only or full control failure

7. Stop if Resolved

  • If the display returns to normal after reset, perform a functional test
  • Verify proper operation before returning to service

If the Problem Persists

If Error 52 continues after completing all external troubleshooting steps, internal communication between the display and control board has likely failed. This condition cannot be corrected externally.

  • Remove the device from service
  • Label the device as Out of Service
  • Send the unit for repair or board-level evaluation

Clinical Use Tip

Never troubleshoot on an active patient. Move patient to backup device first.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Pump displaying Error 52 with frozen or unresponsive screen during feeding."

Cause

What was observed during troubleshooting.

Example:
"Display/control board communication failure not resolved by power cycle."

Resolution

What action was taken.

Example:
"Performed power cycle and inspection; issue persisted. Removed device from service and sent for repair."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Error 52 is a strong indicator of internal display communication failure. Always rule out simple external causes such as power issues or transient faults, but recognize when escalation is required to maintain safe and reliable patient care.

That is successful troubleshooting.
