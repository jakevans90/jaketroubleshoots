Baxter Novum IQ

Drug Library Not Loading or Corrupted

Asset Type

Infusion Pump

Manufacturer

Baxter

Model

Novum IQ

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting situations where the Novum IQ infusion pump fails to load the drug library, displays errors related to the drug library, or appears to have missing/corrupted medication profiles. The focus is on external causes such as configuration, network sync, and user setup before assuming internal failure.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Do not troubleshoot while the pump is actively infusing a patient.
  • Move the patient to a properly functioning pump before proceeding.

Verify the Reported Issue

  • Power on the pump and attempt to access the drug library.
  • Note any error messages (e.g., “Library Not Found,” “Library Corrupt,” or sync errors).
  • Confirm whether the issue is consistent or intermittent.

Why this matters: Confirms whether it is a true library issue versus user workflow confusion.

Check Correct Care Area/Profile Selection

  • Ensure the correct care unit or profile is selected on the pump.
  • Some libraries are unit-specific and may appear missing if the wrong profile is chosen.

Why this matters: Misconfiguration can mimic a missing or corrupted library.

Confirm Network Connectivity

  • Verify the pump is connected to Wi-Fi or the hospital network.
  • Check for network indicators or connection status on the device.
  • Move the pump to a known good coverage area if needed.

Why this matters: Drug libraries are often delivered or updated via the network.

Force Library Sync / Refresh

  • Attempt to manually sync or refresh the drug library from the settings menu.
  • Reboot the pump and allow time for automatic synchronization.

Why this matters: Temporary sync failures can prevent the library from loading properly.

Check Server / Integration Status

  • Confirm with IT or clinical systems that the drug library server or integration platform is operational.
  • Verify there are no ongoing outages or deployment issues.

Why this matters: A system-wide issue can affect multiple pumps simultaneously.

Compare with Another Pump

  • Check a known working Novum IQ pump on the same unit.
  • Verify if the drug library loads correctly on that device.

Why this matters: Helps determine if the issue is device-specific or system-wide.

Inspect for Incomplete Updates

  • Determine if a recent drug library update was pushed.
  • Look for signs of interrupted updates (battery loss during update, network drop).

Why this matters: Partial updates can result in corrupted or unusable libraries.

Check Date and Time Settings

  • Ensure the system date and time are correct.

Why this matters: Incorrect time settings can interfere with validation or syncing of drug libraries.

Perform a Controlled Reboot

  • Power cycle the pump fully and allow it to reconnect and resync.

Why this matters: Clears temporary software faults affecting library loading.

If the Problem Persists

If the drug library still does not load or appears corrupted after completing the above steps, common external causes have been ruled out. The issue is likely related to internal software corruption or configuration failure.

  • Remove the device from service
  • Label the device Out of Service
  • Send for repair or bench evaluation (possible reimage, software reload, or vendor support)

Knowing when to stop prevents unnecessary risk and ensures patient safety.

Clinical Use Tip

Do not attempt to troubleshoot or reload the drug library while the pump is actively infusing a patient. Transition the patient to a properly functioning pump before proceeding. A missing or incorrect drug library can lead to dosing errors.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“Nursing reported infusion pump drug library would not load; pump displayed “Library Not Found” and could not start medication using guardrails.”

Cause

What was observed during troubleshooting.

Example:
“Pump was not connected to the network and failed to sync the drug library. After reconnecting, library still failed to load, suggesting incomplete or corrupted update.”

Resolution

What action was taken.

Example:
“Attempted network reconnection, forced sync, and reboot with no success. Removed pump from service and sent to vendor for software reload.”

Helpful Details to Include (If Known)

  • Network connection status verified
  • Sync attempt results
  • Error messages displayed
  • Comparison with known working pump
  • Recent update activity
  • Date/time accuracy
  • Final device status (removed from service or returned to use)

Final Thought

Drug library functionality is critical for safe infusion practices. Always start with simple configuration and connectivity checks before assuming failure. When the integrity of the drug library is in question, removing the device from service is the correct decision. Clear documentation ensures traceability and supports system-wide improvements.

That is successful troubleshooting.
