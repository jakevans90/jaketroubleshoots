Drager Evita V500

Alarm: Device check required – configuration change or system reset detected

Asset Type

Ventilator

Manufacturer

Drager

Model

Evita V500

What This Guide Helps With

This guide addresses the “Device check required” alarm, which indicates the ventilator has undergone a configuration change or system reset and cannot verify safe operation until a full device check is completed.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Confirm the patient is supported by an alternate ventilator if this alarm prevents use
  • Remove the device from active patient use if necessary
  • Transition to backup ventilation immediately

Verify the Reported Alarm and Behavior

  • Power on the ventilator and confirm “Device check required” is displayed
  • Observe when the alarm appears (startup, after reset, or configuration change)
  • Check whether ventilation is inhibited or limited

Check for Recent Configuration Changes or Reset Conditions

  • Determine if the ventilator experienced a recent power loss or restart
  • Check for recent software or configuration changes
  • Verify if a new circuit or accessory was connected

Verify External Setup Before Running Device Check

  • Ensure the ventilator is connected to stable AC power
  • Confirm gas supplies (air and O2) are connected and within normal pressure range
  • Verify the breathing circuit is properly assembled and connected
  • Inspect for visible damage or contamination

Perform Full Device Check (System Test)

  • Initiate the full device check following normal startup procedure
  • Follow on-screen prompts completely
  • Allow all test phases to complete without interruption

Evaluate Device Check Results

  • If the device check passes, confirm the alarm clears and the unit is ready for use
  • If the device check fails, document any error messages
  • Do not attempt to use the ventilator on a patient if testing fails

If the Problem Persists

If the ventilator continues to require a device check or fails testing:

  • All external setup issues have been ruled out
  • This indicates a potential internal system fault, sensor issue, or valve failure
  • Remove the ventilator from service
  • Label the unit Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot or attempt to clear this alarm on an active patient. Always move the patient to a verified ventilator before performing device checks.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Ventilator displaying 'Device check required' at startup; unable to place into service."

Cause

What was observed during troubleshooting.

Example:
"System detected configuration change or reset requiring full device validation; device check pending or failed."

Resolution

What action was taken.

Example:
"Performed full device check. Unit passed and returned to service OR failed testing and was removed from service for repair. Verified power, gas supply, and circuit prior to testing."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

A “Device check required” alarm is a safety lockout condition. The ventilator is preventing use until it verifies proper function. Always validate setup, complete the system test, and escalate if the device cannot confirm safe operation.

That is successful troubleshooting.
