GE Healthcare MAC 5

Error: Software Image Not Found in Shared Folder

Asset Type

Electrocardiograph (EKG) Machine

Manufacturer

GE Healthcare

Model

MAC 5

What This Guide Helps With

This guide helps troubleshoot Software Image Not Found in Shared Folder on a GE Healthcare MAC 5 when the device cannot locate a valid software image in the configured network shared path.

This condition can prevent software update activity from continuing and may indicate a missing file, incorrect shared folder path, invalid software image, network access problem, or shared folder permission issue.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Remove the MAC 5 from patient use before troubleshooting software update or software loading issues.
  • Do not attempt software update troubleshooting while the device is needed for active ECG acquisition.
  • Move patient ECG testing to another available ECG machine if needed.
  • Expected: Patient ECG testing is protected before software image troubleshooting begins.

Verify the Reported Error and Behavior

  • Confirm the displayed message is exactly: Software Image Not Found in Shared Folder.
  • Observe whether the error appears at startup, during a software update attempt, when accessing a shared network location, after changing network settings, or after replacing or moving software files.
  • Expected: The error is confirmed and tied to the software image retrieval process.

Translate the Error Into the Real-World Problem

  • This error means the MAC 5 is looking in a configured shared folder for a software image, but it does not find a valid image file there.
  • Likely causes include an incorrect shared folder path, missing software image, wrong file copied to the folder, invalid or unsigned software image, network access issue, permission issue, credential issue, or incorrect shared path configuration.
  • Expected: Troubleshooting focuses on file location, file validity, and network share access.

Confirm the Device Is on a Stable Power Source

  • Connect the MAC 5 to AC power before continuing.
  • Do not perform software image checks or update-related work on battery power alone.
  • Expected: The MAC 5 remains powered and stable during troubleshooting.

Confirm Basic Network Connectivity

  • Verify whether the MAC 5 is connected by LAN or WLAN.
  • For LAN, confirm the Ethernet cable is fully seated, check for link or activity lights if available, and try a known-good network cable and wall jack if possible.
  • For WLAN, confirm the device is connected to the correct wireless network, check signal strength, and move closer to an access point if the issue is intermittent.
  • Expected: The MAC 5 has reliable network connectivity before checking the shared folder.

Verify the Shared Path

  • Confirm the shared path entered on the MAC 5 matches the intended network location exactly.
  • Check for an incorrect server name, wrong folder name, missing subfolder, typographical error, old share path from a previous update process, or path changed by IT or Clinical Engineering.
  • Expected: The configured shared path points to the correct folder.

Confirm the Software Image Is Actually Present

  • From a workstation with access to the same share, open the shared folder and verify that the software image file is present.
  • Confirm the folder is not empty and that the file was not placed in a different subfolder.
  • Expected: The required software image is visible in the configured shared folder.

Confirm the Software Image Is Valid and Digitally Signed

  • Verify that the file placed in the shared folder is the correct GE Healthcare MAC 5 software image and not a renamed, incomplete, extracted, or unrelated file.
  • Make sure the software image is valid and digitally signed as required.
  • Expected: The shared path contains a valid digitally signed software image.

Check for File Transfer or Copy Problems

  • If the software image was recently copied to the shared folder, confirm the file copy completed successfully.
  • Check for partial file transfer, zero-byte file, interrupted copy, compressed archive placement, wrong version, or security software quarantine.
  • Expected: The software image file is complete and accessible.

Confirm Shared Folder Permissions

  • Verify the MAC 5 or the account used by the MAC 5 has permission to access the shared folder.
  • Confirm read permission exists, credentials are current, the password has not expired, and both share permissions and NTFS permissions allow access.
  • Check with IT or the system administrator if needed.
  • Expected: The MAC 5 can read the shared folder and software image.

Check Whether Other Devices Can Access the Same Image

  • If multiple MAC 5 units use the same shared software image location, test whether another MAC 5 can see the image.
  • If another MAC 5 works from the same shared folder, the issue may be isolated to this unit’s network configuration, credentials, or local software state.
  • If no MAC 5 can see the image, the issue is likely with the shared folder, file placement, permissions, or image validity.
  • Expected: The issue is narrowed to one device or the shared software location.

Restart and Recheck After Corrections

  • After correcting the path, file placement, network access, or permissions, restart the MAC 5 and attempt the process again.
  • Expected: The MAC 5 locates the software image without displaying the error.
  • If resolved, document the correction and return the device to service only after normal operation is confirmed.

Stop When Resolved

  • If the valid digitally signed software image is placed in the correct shared path and the MAC 5 recognizes it, no further troubleshooting is needed.
  • Confirm the device boots normally, network settings remain correct, and ECG acquisition is available before returning it to service.

If the Problem Persists

If the shared path is correct, the software image is valid and digitally signed, network access is confirmed, permissions are correct, and the error still appears, external causes have been ruled out.

Remove the MAC 5 from service, label it Out of Service, and send it for repair, bench evaluation, IT/network support, or GE service support as appropriate.

Persistent software image detection errors may indicate a device software, configuration, storage, network authentication, or update process issue that requires higher-level support.

Clinical Use Tip

Never troubleshoot software update errors on an active patient. If the MAC 5 is needed for ECG testing, move the patient workflow to a backup ECG machine first and maintain therapy or diagnostic continuity until the software image issue is resolved and normal operation is verified.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"User reported MAC 5 displays Software Image Not Found in Shared Folder during software image access or update attempt. Device could not locate update image in configured shared path."

Cause

What was observed during troubleshooting.

Example:
"Shared folder did not contain a valid digitally signed software image, or the device could not access the correct shared path due to path, permission, credential, or network configuration issue."

Resolution

What action was taken.

Example:
"Verified AC power, network connection, shared path, folder access, and software image placement. Confirmed valid digitally signed software image was present in the correct shared folder. Restarted device and verified error cleared. Returned device to service."

Helpful Details to Include (If Known)

  • Alarm behavior or exact error behavior
  • Accessories swapped, including network cable, wall jack, USB media, or workstation access path if applicable
  • Power behavior and whether AC power was connected during troubleshooting
  • Environmental factors, including network location, wireless signal strength, or recent device movement
  • Indicator lights, including LAN link/activity lights or wireless connection status
  • Final device status, including whether the error cleared, whether ECG acquisition was verified, and whether the device was returned to service or removed from service

Final Thought

Software image errors should be handled carefully because they may affect update readiness and device availability. Start with patient safety, then verify the shared path, network access, permissions, and the presence of a valid digitally signed software image before escalating. Clear CCR documentation helps separate a simple file-location issue from a true device or software failure.

That is successful troubleshooting.
