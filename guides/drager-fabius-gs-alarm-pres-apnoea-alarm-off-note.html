Drager Fabius GS

Alarm: PRES APNOEA ALARM OFF !

Asset Type

Anesthesia Machine

Manufacturer

Drager

Model

Fabius GS

What This Guide Helps With

PRES APNOEA ALARM OFF ! alarm indicating pressure apnea alarms are disabled in ManSpont mode, reducing safety monitoring and increasing risk of undetected loss of ventilation.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Do not troubleshoot while connected to a patient if monitoring is compromised
  • Ensure alternate monitoring or ventilation support is available
  • Transition patient if alarms are not active

2. Verify the Reported Alarm and Behavior

  • Confirm “PRES APNOEA ALARM OFF !” is displayed
  • Check if the unit is in ManSpont mode
  • Verify that pressure/apnea alarms are not active

3. Check Ventilation Mode

  • Confirm the machine is in ManSpont (Manual/Spontaneous) mode
  • Understand this alarm is mode-dependent

4. Activate Pressure Apnea Alarm Settings

  • Navigate to alarm configuration settings
  • Enable pressure apnea alarms or pressure monitoring alarms
  • Ensure settings are applied

5. Verify Alarm Functionality

  • Confirm alarm limits are set appropriately
  • Observe active pressure monitoring and alarm readiness
  • Ensure alarm message clears

6. Check for User Configuration Cause

  • Confirm with staff if alarms were intentionally disabled
  • Reinforce proper alarm usage and safety practices

7. Confirm Resolution

  • Ensure alarm is cleared
  • Verify monitoring is active and functional
  • Return device to service if safe

If the Problem Persists

If the alarm remains after enabling pressure alarms, external causes have been ruled out. This may indicate a configuration or interface issue.

  • Remove device from service
  • Label Out of Service
  • Send for repair or further evaluation

Clinical Use Tip

Never troubleshoot on an active patient. Always ensure apnea and pressure alarms are active before use and move the patient to a backup device if monitoring is compromised.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Device displaying PRES APNOEA ALARM OFF ! during setup."

Cause

What was observed during troubleshooting.

Example:
"Pressure apnea alarms were disabled in system settings while in ManSpont mode."

Resolution

What action was taken.

Example:
"Re-enabled pressure apnea alarms, verified alarm functionality, and confirmed safe monitoring."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Some alarms indicate reduced safety rather than device failure. Recognizing and correcting configuration-based issues is critical to maintaining proper patient monitoring.

That is successful troubleshooting.
