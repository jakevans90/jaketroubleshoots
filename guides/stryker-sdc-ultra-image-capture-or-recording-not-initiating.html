Stryker SDC Ultra

Image Capture or Recording Not Initiating

Asset Type

Endoscopic / OR Camera System

Manufacturer

Stryker

Model

SDC Ultra

What This Guide Helps With

Failure to start image capture or recording due to storage, connection, control input, or system readiness issues.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • If recording is required for documentation during a procedure, notify clinical staff and transition to a backup documentation method if available.
  • Expected: Procedure continues safely without delay
  • Why it matters: Recording issues should not interfere with patient care or workflow

Verify the Reported Issue

  • Attempt to initiate image capture or recording using the touchscreen, footswitch, or control interface.
  • Observe:
    • No response when pressing record/capture
    • Error messages on screen
    • Recording button grayed out or inactive
  • Expected: Issue is reproducible and clearly identified
  • Why it matters: Confirms a true system issue vs user workflow or timing

Check Storage Media (Most Common Cause)

  • Verify that a USB drive or approved storage device is inserted and recognized by the system.
  • Action:
    • Remove and reinsert USB device
    • Try a known-good USB device
    • Check for “storage full” or “no media” messages
  • Expected: System detects storage and enables recording
  • Why it matters: The SDC Ultra will not record without valid, writable storage

Verify Storage Capacity and Format

  • Check if the storage device is full or improperly formatted.
  • Action:
    • Confirm available storage space
    • Use a properly formatted device (typically FAT32 or system-compatible format)
    • Replace with a cleared or new USB drive
  • Expected: Recording initiates once valid storage is available
  • Why it matters: Full or incompatible media prevents recording initiation

Check Input Signal from Video Source

  • Confirm that the SDC Ultra is receiving a valid video signal from the camera system.
  • Action:
    • Verify video is displayed on the monitor
    • Check input cables (DVI/HDMI/SDI depending on setup)
    • Reseat or swap cables if needed
  • Expected: Active video signal present
  • Why it matters: Recording cannot start without an active input signal

Check Control Method (Touchscreen / Footswitch)

  • Verify the control interface being used to initiate recording.
  • Action:
    • Test recording via touchscreen instead of footswitch
    • Inspect footswitch connection and cable
    • Swap with known-good footswitch if available
  • Expected: Recording starts using alternate control method
  • Why it matters: Faulty accessories can prevent command input

Confirm System is Not in Error or Standby State

  • Check for system messages or modes that prevent recording.
  • Action:
    • Look for error notifications
    • Ensure system is fully booted and not in standby
    • Clear any prompts or warnings
  • Expected: System ready state allows recording
  • Why it matters: Certain system states disable recording functionality

Restart the System

  • Perform a full system reboot.
  • Action:
    • Power down the SDC Ultra
    • Wait 10–15 seconds
    • Power back on and retest
  • Expected: Recording function restored after reboot
  • Why it matters: Clears temporary software or interface faults

If the Problem Persists

If storage, signal input, control interfaces, and system state have been verified, the issue is likely internal (software fault, interface board failure, or control module issue).

Remove the device from service, label it Out of Service, and send for repair or bench evaluation.

Clinical Use Tip

Do not troubleshoot recording issues during an active procedure. Ensure the patient is supported with an alternate system before testing functionality.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"SDC Ultra not initiating recording during procedure."

Cause

What was observed during troubleshooting.

Example:
"USB storage device was full and not allowing new recordings."

Resolution

What action was taken.

Example:
"Replaced with cleared USB device and verified recording functionality."

Helpful Details to Include (If Known)

  • USB device tested and recognized
  • Storage capacity checked
  • Video signal confirmed present
  • Footswitch and touchscreen tested
  • No active system errors
  • System reboot performed
  • Final device status verified operational or removed from service

Final Thought

Start with storage and signal verification, as these are the most common causes. Maintain a structured approach and avoid unnecessary escalation. Proper documentation ensures repeat issues are quickly resolved.

That is successful troubleshooting.
