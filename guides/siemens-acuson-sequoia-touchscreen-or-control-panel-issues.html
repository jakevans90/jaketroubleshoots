Siemens Healthineers ACUSON Sequoia

Touchscreen or Control Panel Issues

Asset Type

Ultrasound System

Manufacturer

Siemens Healthineers

Model

ACUSON Sequoia

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting situations where the ACUSON Sequoia touchscreen, control panel, or keyboard becomes unresponsive, erratic, or intermittently fails. Common presentations include:

  • Touchscreen does not respond to finger input.
  • Buttons or control panel keys do not register.
  • Cursor or menu navigation is delayed, jumps, or misbehaves.

The focus is on external, simple checks before escalating to internal repair.

Step-by-Step Troubleshooting

Verify Power Connection

  • Ensure the system is properly plugged into a tested, live electrical outlet.
  • Check for loose or damaged power cords.
  • Why: Low or intermittent power can cause control electronics to behave erratically.

Check System Boot and Status Lights

  • Observe startup indicators and error messages.
  • Confirm the system completes booting without alerts related to control panel or touchscreen.
  • Why: Some touchscreen or panel issues appear only during or after boot errors.

Inspect Touchscreen and Control Panel Surface

  • Clean the screen and buttons with approved non-abrasive cleaner.
  • Remove any protective covers or films that could interfere with capacitive touch.
  • Why: Debris or moisture can prevent proper input registration.

Test Input with Alternate Method

  • If available, connect a mouse or use keyboard shortcuts to navigate menus.
  • Observe if control issues persist when bypassing touchscreen or panel.
  • Why: Helps isolate whether the problem is the display/panel or the main system.

Check for Loose or Damaged Cables

  • Inspect ribbon or connection cables between the touchscreen/control panel and main system.
  • Re-seat cables if accessible without disassembly.
  • Why: Intermittent contact can mimic failure.

Perform a System Restart

  • Power down the system completely, wait 30–60 seconds, then power on.
  • Observe if touchscreen responsiveness improves.
  • Why: Temporary software glitches can affect input responsiveness.

Verify Software Updates and Calibration

  • Check if the system requires touchscreen calibration (if available in menu).
  • Confirm system software is up to date.
  • Why: Software inconsistencies can affect panel behavior.

If the Problem Persists

Common external causes have been ruled out.

  • Remove the system from service.
  • Label Out of Service and send for bench evaluation or manufacturer repair.
  • Remember: Knowing when to escalate is proper troubleshooting.

Clinical Use Tip

  • Do not attempt touchscreen or panel troubleshooting while the device is in use on a patient.
  • Move patients to an alternate unit if imaging is needed urgently.
  • Troubleshoot only when patient safety can be ensured.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“Touchscreen and control panel buttons intermittently unresponsive.”

Cause

What was observed during troubleshooting.

Example:
Observed touchscreen and panel do not register input; no debris or loose cords found; system software up to date.

Resolution

What action was taken.

Example:
Device labeled Out of Service; sent for bench evaluation with Siemens Healthineers service.

Helpful Details to Include

  • Outlet tested and verified live.
  • Power cord inspected.
  • Startup indicators and error messages noted.
  • Screen and panel cleaned and visually inspected.
  • Alternate input method tested.
  • Calibration and software version noted.
  • Final device status: Out of Service pending repair.

Final Thought

Patient safety and proper escalation are paramount. Always start with simple, external checks before assuming internal failure. Document each step in the CCR format to support clear communication and cost-effective resolution.

That is successful troubleshooting.
