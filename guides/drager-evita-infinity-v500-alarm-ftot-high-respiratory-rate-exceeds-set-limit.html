Drager Evita V500

Alarm: fTOT high Respiratory rate exceeds set limit

Asset Type

Ventilator

Manufacturer

Drager

Model

Evita V500

What This Guide Helps With

This guide helps troubleshoot the fTOT high alarm, which indicates the total respiratory rate is above the set limit. This can result from patient distress, improper settings, or false triggering, and may lead to ineffective ventilation or patient-ventilator mismatch.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Assess the patient’s respiratory status immediately
  • Provide alternate ventilation if needed before troubleshooting
  • Confirm adequate oxygenation and ventilation

Verify the Reported Alarm and Behavior

  • Confirm “fTOT high” is active on the display
  • Observe total respiratory rate value
  • Note if alarm is continuous or intermittent
  • Differentiate spontaneous vs mandatory breaths

Check Alarm Limit Settings (Most Common Cause)

  • Review fTOT high alarm limit setting
  • Compare with expected clinical range
  • Check for recent adjustments by staff

Assess Patient Contribution to Rate

  • Observe for rapid breathing or distress
  • Check for pain, anxiety, or hypoxia
  • Coordinate with clinical staff

Check Ventilator Mode and Settings

  • Review ventilation mode (e.g., SIMV, pressure support)
  • Check trigger sensitivity and backup rate
  • Verify no unintended auto-triggering

Inspect Circuit and Trigger Sources

  • Check for leaks in the breathing circuit
  • Inspect for water accumulation in tubing
  • Ensure flow sensors are clean and properly connected

Perform System Test if Needed

  • Run System Self Test (SST)
  • Run Extended System Test (EST) if available
  • Confirm device passes all tests

If the Problem Persists

If all external causes have been ruled out and the alarm continues, an internal sensing or processing issue may be present.

  • Remove the ventilator from service
  • Label the device as Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot while the ventilator is actively supporting an unstable patient. Always transition the patient to a backup device before making adjustments or performing checks.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Ventilator alarming fTOT high with elevated respiratory rate during use."

Cause

What was observed during troubleshooting.

Example:
"Respiratory rate exceeded alarm limit due to patient over-breathing or alarm limits set too low."

Resolution

What action was taken.

Example:
"Adjusted alarm limits and verified ventilator settings; confirmed normal operation or removed device from service if unresolved."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

High respiratory rate alarms are commonly caused by patient condition or configuration issues. Always rule out external and clinical factors before suspecting device failure, and prioritize patient safety and proper escalation.

That is successful troubleshooting.
