Smiths Medical Medfusion 4000

Drug Library or Configuration Corruption

Asset Type

Infusion Pump

Manufacturer

Smiths Medical

Model

Medfusion 4000

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting issues where the Medfusion 4000 infusion pump fails to access or correctly apply its drug library or configuration settings. Symptoms may include missing drugs, incorrect infusion parameters, or errors when loading therapy profiles. Most often, these issues are caused by corrupted software files, interrupted updates, or external storage/media problems rather than hardware failure.

Step-by-Step Troubleshooting

Verify Device Status

  • Ensure the pump is powered on and completes its self-test without alarms.
  • Check for error messages related to the drug library or configuration.

Why: Confirms the device is in a stable state before making changes.

Check External Media or Network Connection

  • If updates are applied via USB, SD card, or network, ensure the media is properly inserted and readable.
  • Inspect for damage, dirt, or loose connections.

Why: Corrupted media can prevent proper library loading.

Confirm Device Software Version

  • Navigate to the system info screen and note the software and drug library version.
  • Compare with the hospital’s approved version list.

Why: Outdated or mismatched versions can appear as corruption.

Attempt Library Reload or Configuration Restore

  • Use the manufacturer-approved process to reload the drug library or restore default configuration.
  • Ensure the pump is on AC power during this operation.

Why: Often resolves minor corruption without hardware intervention.

Check Logs for Errors

  • Access event logs or error history on the pump.
  • Look for failed update attempts, read/write errors, or repeated library load failures.

Why: Helps identify whether the issue is persistent or transient.

If the Problem Persists

If the drug library or configuration cannot be restored, or repeated errors occur:

  • Remove the device from service.
  • Label it Out of Service.
  • Send for vendor repair or bench evaluation.

Why: Continuing to use a corrupted device risks patient safety and therapy errors.

Clinical Use Tip

Do not attempt to infuse medications on a pump showing drug library or configuration errors. Transfer the patient to another verified pump before troubleshooting. Only reload libraries or restore configurations when the device is off patient and stable.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“Pump shows missing drug profiles and errors when loading therapy.”

Cause

What was observed during troubleshooting.

Example:
“External media checked; pump failed to reload drug library; configuration appears corrupted.”

Resolution

What action was taken.

Examples:
“Attempted reload using manufacturer-approved procedure; issue persists. Device labeled Out of Service and sent for repair.”

Helpful Details to Include

  • AC power confirmed during reload attempt
  • Media checked and swapped if applicable
  • Software and drug library versions noted
  • Error messages recorded
  • Device labeled Out of Service

Final Thought

Maintaining accurate drug libraries and configurations is critical for patient safety. Troubleshooting must follow a logical sequence: check external and easily corrected causes first, escalate when restoration fails, and always document clearly using the CCR method.

That is successful troubleshooting.
