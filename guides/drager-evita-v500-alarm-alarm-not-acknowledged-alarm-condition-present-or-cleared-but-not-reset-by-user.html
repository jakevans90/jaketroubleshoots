Drager Evita V500

Alarm: Alarm not acknowledged Alarm condition present or cleared but not reset by user

Asset Type

Ventilator

Manufacturer

Drager

Model

Evita V500

What This Guide Helps With

This guide addresses the alarm condition where the ventilator indicates “Alarm not acknowledged,” meaning an alarm condition is still active or was cleared but not manually reset. This results in continued audible and visual alerts and can cause confusion or delayed response to real patient issues.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Confirm the patient is being properly ventilated
  • Verify adequate oxygenation and chest rise
  • Have backup ventilation available if needed

2. Verify the Reported Alarm and Behavior

  • Confirm the ventilator displays “Alarm not acknowledged”
  • Observe if any alarm conditions are still present
  • Check if the alarm continues despite normal readings

3. Check for Active Alarm Conditions

  • Review the alarm list on the display
  • Identify unresolved alarms (pressure, volume, FiO2, etc.)
  • Correct any active alarm conditions

4. Acknowledge and Reset the Alarm

  • Press the ALARM RESET button
  • Confirm acknowledgment using the rotary control if prompted
  • Ensure audible and visual alarms stop

5. Check User Interface Responsiveness

  • Verify buttons and rotary control are functioning
  • Attempt acknowledgment again
  • Observe for delayed or no response

6. Power Cycle the Ventilator (If Safe to Do So)

  • Remove the device from patient use
  • Power cycle the ventilator
  • Allow full startup sequence to complete

7. Stop When Resolved

  • Return the ventilator to service if alarm clears
  • Verify normal operation
  • Monitor for recurring alarms

If the Problem Persists

If the alarm cannot be acknowledged or continues without an active condition:

  • External causes have been ruled out
  • This suggests a user interface or internal system issue
  • Remove the device from service
  • Label Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot alarms on an active patient. Always move the patient to a backup ventilator before attempting alarm acknowledgment or system checks.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Ventilator displaying ‘Alarm not acknowledged’ with continuous audible alarm despite no visible active alarm condition."

Cause

What was observed during troubleshooting.

Example:
"Alarm condition had cleared but was not manually acknowledged, or user interface input delay prevented reset."

Resolution

What action was taken.

Example:
"Reviewed alarm list, confirmed no active alarms, pressed ALARM RESET and confirmed with rotary control. Alarm cleared and device tested."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

Alarm acknowledgment is a critical safety function that ensures clinician awareness of patient conditions. Proper troubleshooting verifies whether the issue is user action, system behavior, or a deeper fault requiring escalation.

That is successful troubleshooting.
