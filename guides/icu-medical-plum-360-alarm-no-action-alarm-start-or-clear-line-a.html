ICU Medical Plum 360

Alarm: No Action Alarm! Start or Clear Line A.

Asset Type

Infusion Pump

Manufacturer

ICU Medical

Model

Plum 360

What This Guide Helps With

This guide addresses the “No Action Alarm! Start or Clear Line A.” condition, which occurs when no operator action is taken after stopping Line A. This prevents infusion from continuing until user input is provided.

Step-by-Step Troubleshooting

1. Ensure Patient Safety First

  • Confirm whether Line A infusion is required for therapy
  • If therapy is needed, resume infusion or transfer to a backup pump
  • Verify the patient continues receiving prescribed treatment

2. Verify the Reported Alarm and Behavior

  • Confirm the alarm message is displayed exactly as shown
  • Check if Line A was previously stopped or paused
  • Determine if the pump has been idle for approximately 2 minutes
  • Verify that Line A infusion is not currently running

3. Acknowledge the Alarm

  • Press any key to acknowledge the alarm
  • Ensure the alarm clears or silences
  • Confirm the pump allows further input

4. Determine Intended Operation for Line A

  • If infusion should continue, press START for Line A
  • If infusion is no longer needed, clear Line A programming
  • Confirm correct action based on clinical intent

5. Check for Workflow or User Setup Issues

  • Verify staff are completing programming steps fully
  • Ensure Line A is not unintentionally left stopped
  • Assess for delays in workflow that may trigger timeout

6. Verify Normal Operation After Action

  • Confirm infusion resumes properly if restarted
  • Ensure no additional alarms are present
  • Monitor for stable pump operation

If the Problem Persists

If the alarm occurs repeatedly without a clear cause:

  • External and user-related causes have been ruled out
  • Persistent alarms may indicate a control or interface issue

Action:

  • Remove the device from service
  • Label the device Out of Service
  • Send for repair or bench evaluation

Clinical Use Tip

Never troubleshoot on an active patient. Always confirm infusion is actively running after any pause or alarm condition before leaving the device in use.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"No Action Alarm displayed and Line A infusion was not running."

Cause

What was observed during troubleshooting.

Example:
"Line A was stopped with no operator input for the timeout period, triggering inactivity alarm."

Resolution

What action was taken.

Example:
"Acknowledged alarm and restarted Line A infusion; verified normal operation."

Helpful Details to Include (If Known)

  • Alarm behavior
  • Accessories swapped
  • Power behavior
  • Environmental factors
  • Indicator lights
  • Final device status

Final Thought

This alarm is designed to prevent unintended idle states rather than indicate a failure. Recognizing workflow-driven alarms helps maintain safe therapy while avoiding unnecessary equipment downtime.

That is successful troubleshooting.
