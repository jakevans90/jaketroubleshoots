Getinge Magnus Series

Table Movement Functions Inoperative

Asset Type

Surgical Table

Manufacturer

Getinge

Model

Magnus Series

What This Guide Helps With

Identifies causes of complete loss of table movement, including power, control input, lockout conditions, and connection issues before internal failure is assumed.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Confirm the table is not actively supporting a patient.
  • If in use, safely transfer the patient to another table before troubleshooting.

Verify Main Power Status

  • Check that the table is plugged into a known working outlet.
  • Confirm power indicator lights are present on the base.
  • If no power indicators → suspect power supply or cord issue.

Check Emergency Stop / Lockout Condition

  • Inspect for an engaged emergency stop or system lock.
  • Some Magnus tables disable movement if safety lockouts are active.
  • Reset if applicable and retest.

Confirm Battery Charge Level

  • If running on battery, verify the battery is charged.
  • Connect to AC power and allow a few minutes, then retry movement.
  • A depleted battery can prevent all motion functions.

Test Multiple Control Inputs

  • Attempt movement using:
  • Hand control (pendant)
  • IR remote (if equipped)
  • Backup control panel on the table base
  • If one control works → issue is isolated to the non-working controller
  • If none work → continue troubleshooting

Inspect Hand Control / Pendant Connection

  • Check that the pendant cable is fully seated and not damaged.
  • Look for bent pins, frayed wiring, or loose connectors.
  • Reseat connection and retest

Check for IR Remote Issues (if applicable)

  • Verify batteries in the IR remote.
  • Ensure there is a clear line of sight to the receiver.
  • Confirm no obstruction or bright light interference.

Listen for Motor or Relay Activity

  • Attempt a movement command and listen:
  • If motors or relays engage but no movement → possible mechanical or actuator issue
  • If no sound at all → likely control or power distribution issue

Inspect for Obvious Mechanical Obstruction

  • Visually check for:
  • Debris under the table
  • Binding joints
  • Accessories interfering with movement
  • Remove obstruction if found and retest

Check System Status Indicators or Error Messages

  • Observe for fault codes, blinking LEDs, or alarm tones.
  • Document any error indications for further diagnosis or escalation.

Power Cycle the Table

  • Turn the table off (if possible), disconnect from AC power, wait 30 seconds, then reconnect and power on.
  • This can reset control system faults
  • If movement is restored → monitor for recurrence

If the Problem Persists

If all external power, control, and obstruction causes have been ruled out, the issue is likely internal (actuator failure, control board, or power distribution fault).

Remove the table from service, label it Out of Service, and send for repair or bench evaluation.

Recognizing when the issue is beyond external troubleshooting is essential to safe and effective Clinical Engineering practice.

Clinical Use Tip

Never attempt to troubleshoot table movement while a patient is on the table.

Always transfer the patient first to prevent injury from unexpected movement or instability.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"Table not responding to any movement commands from hand control or remote."

Cause

What was observed during troubleshooting.

Example:
"Loose hand control connection at base prevented control signal transmission."

Resolution

What action was taken.

Example:
"Reseated hand control connector and verified full table movement functionality restored."

Helpful Details to Include (If Known)

  • Outlet tested and verified
  • Battery charge status
  • Multiple control inputs tested
  • Indicator lights or fault codes observed
  • Motor/relay sound present or absent
  • Any obstruction found
  • Final operational status

Final Thought

Effective troubleshooting starts with simple, external checks and progresses logically. Ensuring patient safety, verifying power and control inputs, and knowing when to escalate prevents unnecessary risk and downtime. Clear documentation supports future reliability and faster repairs.

That is successful troubleshooting.
