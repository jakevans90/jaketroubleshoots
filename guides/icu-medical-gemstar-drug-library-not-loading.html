ICU Medical GemStar

Drug Library Not Loading

Asset Type

Infusion Pump

Manufacturer

ICU Medical

Model

GemStar

What This Guide Helps With

This guide assists Clinical Engineering in troubleshooting ICU Medical GemStar infusion pumps that fail to load or display their drug library. This issue can prevent proper infusion programming and may be caused by simple external factors such as power interruptions, connectivity issues, or data corruption. The guide emphasizes external checks before assuming internal software or hardware failure.

Step-by-Step Troubleshooting

Verify Power Source

  • Ensure the pump is connected to a functioning AC outlet or has a fully charged battery.
  • Why: Low or unstable power can prevent the drug library from loading.

Check Network or Data Connections

  • If the pump relies on a central server for its drug library, ensure Ethernet or Wi-Fi connections are secure.
  • Inspect cables and verify connectivity to the hospital network or CCE interface.

Perform a Pump Restart

  • Power cycle the pump using the main power switch.
  • Why: Temporary software glitches can prevent the drug library from loading.

Verify Drug Library File Availability

  • Check that the drug library is present on the pump’s memory or accessible server.
  • Ensure the file is not corrupted.
  • Why: Missing or corrupted library files are a common cause of load failure.

Check for Error Messages

  • Observe the pump’s display for any specific error codes or messages.
  • Note them for documentation or escalation.

Swap or Reload Library (If Applicable)

  • If allowed by policy and device configuration, reload the drug library from a verified backup.
  • Why: Reapplying the library can restore proper functionality.

If the Problem Persists

All external and user-level checks have been completed.

The issue may be internal, such as memory corruption or software failure.

  • Remove the pump from service.
  • Label the pump Out of Service.
  • Send for vendor repair or internal bench evaluation.

Knowing when to escalate ensures patient safety and proper device management.

Clinical Use Tip

  • Do not attempt to program or deliver medication on a pump with a failed drug library.
  • Move the patient to a verified working pump before troubleshooting.
  • Always verify that infusion settings are correct on a functioning device before administration.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
“GemStar pump will not load drug library; displays blank or error message.”

Cause

What was observed during troubleshooting.

Example:
Observed the pump connected to power, network verified, but library failed to load; error message indicates file not found or corrupted.

Resolution

What action was taken.

Example:
Power cycled pump, verified connections, attempted library reload. Device removed from service and sent for bench evaluation when unresolved.

Helpful Details to Include

  • AC outlet tested: Yes/No
  • Battery level: Good/Low
  • Network or server connectivity status
  • Any error codes displayed
  • Attempts at library reload
  • Final device status

Final Thought

Patient safety is paramount. Troubleshooting should follow a logical path from external checks to escalation. Proper documentation ensures continuity and reduces repeat issues. Escalating when appropriate preserves device integrity and keeps patients safe.

That is successful troubleshooting.
