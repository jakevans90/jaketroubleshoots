GE Healthcare MAC 5

Error: Shared Network Folder Is Disabled in Settings Screen

Asset Type

Electrocardiograph (EKG) Machine

Manufacturer

GE Healthcare

Model

MAC 5

What This Guide Helps With

This guide helps troubleshoot the MAC 5 error Shared Network Folder Is Disabled in Settings Screen. This condition can prevent ECG export, archive access, or file transfer through a configured shared network folder while local acquisition, printing, or other send routes may still function.

Step-by-Step Troubleshooting

Ensure Patient Safety First

  • Confirm the MAC 5 is not actively being used to acquire a patient ECG.
  • If the error appears during or after a patient ECG, make sure the ECG is saved locally or printed if clinically needed before changing settings or restarting the device.
  • Expected: Patient ECG data is protected before troubleshooting begins.
  • Why it matters: Network setting changes, restarts, or workflow testing should not risk loss of patient data or interrupt active patient care.

Verify the Reported Error and Behavior

  • Confirm the exact error displayed is Shared Network Folder Is Disabled in Settings Screen.
  • Ask the user when the message appears.
  • Common situations include attempting to export ECGs to a network folder, trying to archive records to a shared location, sending or saving files through a configured network path, or selecting a shared folder destination that is not currently enabled.
  • Expected: The error is reproducible when a shared network folder function is selected.
  • Why it matters: This confirms the issue is tied to disabled shared folder configuration rather than a general network failure.

Confirm the Real-World Impact

  • Determine what function is failing.
  • Check whether the MAC 5 can still acquire ECGs, save ECGs locally, print ECGs, connect to LAN or WLAN, or send by another configured route if available.
  • Expected: The ECG acquisition function may still work, but network shared folder export or archive is blocked.
  • Why it matters: This separates a shared folder setting issue from a full device or network outage.

Check Network Connection First

  • Verify the MAC 5 is connected to the correct network.
  • For wired LAN, check that the Ethernet cable is fully seated, the wall jack is known-good, the network link indicator is present if visible, and the cable or connector is not damaged.
  • For WLAN, check that the correct wireless network is selected, signal strength is acceptable, and the device is not in an area with poor wireless coverage.
  • Expected: The MAC 5 has a valid network connection before shared folder settings are tested.
  • Why it matters: The shared folder setting may be disabled, but the device still needs network access once enabled.

Check the Shared Network Folder Setting

  • Go to the network configuration area and check the shared folder option.
  • Follow the path provided for this error: Network > Shared Network settings.
  • Verify that the shared network path is enabled.
  • Expected: Shared network folder access is enabled in settings.
  • Why it matters: The error directly indicates the network shared path is disabled in the settings screen.

Verify the Shared Path Configuration

  • Confirm the shared network folder information is present and correct.
  • Check the shared folder path, server name or IP address, folder name, username if required, password if required, domain if required by the facility, and correct slash format for the network path.
  • Expected: The shared path is enabled and points to the correct network folder.
  • Why it matters: Enabling the feature alone may not resolve the issue if the stored path or credentials are incorrect.

Compare With a Known-Good MAC 5

  • If another MAC 5 is working on the same network, compare the shared folder settings.
  • Check the shared path format, network destination, LAN or WLAN profile type, and authentication method.
  • Do not copy credentials unless authorized by facility policy.
  • Expected: The affected MAC 5 should match the approved configuration used by working devices.
  • Why it matters: This helps identify a local configuration issue without assuming hardware failure.

Test With a Known-Good Network Location

  • If allowed by site policy, test the MAC 5 from a known-good network jack or known-good wireless area.
  • Retry the shared folder function after confirming the setting is enabled.
  • Expected: The shared folder function proceeds without the disabled setting error.
  • Why it matters: This helps separate configuration failure from a local network access problem.

Restart the MAC 5 After Configuration Changes

  • After enabling or correcting the shared network setting, restart the MAC 5 if the setting does not take effect immediately.
  • Retest the same workflow that originally triggered the error.
  • Expected: The error no longer appears, and the shared folder function is available.
  • Why it matters: Some network or destination settings may not fully refresh until the device restarts.

Stop When Resolved

  • If enabling and configuring the shared network path resolves the error, document the corrected setting and return the MAC 5 to service.
  • Confirm with the user that the intended export, archive, or shared folder workflow now works.
  • Expected: Device is returned to service after successful functional confirmation.

If the Problem Persists

If the shared network folder is enabled and configured but the error or workflow failure continues, external and common configuration causes have been ruled out. This includes network connection checks, LAN or WLAN access, shared folder enablement, shared path review, credentials or permission review, known-good location testing, and comparison with a known-good MAC 5.

Remove the MAC 5 from service if the shared folder function is required for clinical workflow. Label it Out of Service and send it for repair, IT/network escalation, or bench evaluation.

Persistent failure may indicate corrupt network configuration, incorrect permissions on the shared folder, authentication or domain issues, a facility network policy change, or a software/configuration fault on the MAC 5.

Clinical Use Tip

Never troubleshoot on an active patient. Do not troubleshoot network archive settings while the device is attached to a patient or being used for active ECG acquisition. Move the patient to a backup device first if ECG testing is needed, confirm ECGs are saved locally or printed before restarting or reconfiguring, and maintain therapy and diagnostic workflow continuity.

Work Order Documentation (CCR Method)

CCR = Complaint, Cause, Resolution

Complaint

What was reported by the clinical staff.

Example:
"User reported MAC 5 displayed Shared Network Folder Is Disabled in Settings Screen when attempting to use the shared network folder for ECG export or archive."

Cause

What was observed during troubleshooting.

Example:
"Shared network path was disabled under Network > Shared Network settings, preventing the configured shared folder workflow from being used."

Resolution

What action was taken.

Example:
"Verified network connection, enabled and configured shared network path, restarted device, and confirmed shared folder function operated without the error. Returned MAC 5 to service."

Helpful Details to Include (If Known)

  • Alarm behavior, including the exact error text displayed and whether the error occurs during export, archive, or send workflow
  • Accessories swapped, including Ethernet cable, network jack, or alternate approved network connection if tested
  • Power behavior, including whether the device was restarted after configuration changes
  • Environmental factors, including LAN or WLAN area tested, signal strength, and known-good network location used
  • Indicator lights, including network link indicators or connection status messages if visible
  • Final device status, including shared path enabled or disabled, shared folder path reviewed, credentials or permissions suspected, comparison with known-good MAC 5, and whether the device was returned to service or labeled Out of Service

Final Thought

This error points to a configuration condition, not an immediate hardware failure. Start with patient safety, confirm the shared folder behavior, verify network access, then enable and configure the shared network path. If the setting is correct but the device still fails, escalate appropriately and document the failure clearly.

That is successful troubleshooting.
